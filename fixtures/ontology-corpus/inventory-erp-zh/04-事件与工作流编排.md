## 1. 目的

本文件定义进销存全流程的**事件目录** (每个事件: 名称、载荷字段、由哪个动作产生、被哪个动作消费) 与**工作流定义** (把动作按顺序 / 分支 / 事件驱动串成端到端流程,标注触发事件、分支条件、参与角色、编排策略)。事件总线为全企业统一异步通道 (规则 R-40)。命名与 `01-业务流程SOP.md`、`02-业务规则与制度.md`、`03-系统与数据实体.md` 全文一致。

## 3. 工作流定义

### WF-01: 销售到回款工作流 (Order-to-Cash)
- **触发事件:** **报价单已批准**
- **编排策略:** 事件驱动编排 (Event-driven Choreography),跨 CRM/ERP/WMS/FMS。
- **参与角色:** 销售代表、信用专员、拣货员、仓管员、应收财务专员。
- **动作序列:**
  1. `创建销售订单` → 发出 **销售订单已创建**
  2. `执行信用校验` → 分支: 通过 → **信用校验通过**; 失败 → **订单信用冻结** (转 WF-05)
  3. `检查并占用库存` → 分支: 充足 → **库存已占用**; 缺货 → **缺货预警** (转 WF-02)
  4. `生成拣货任务` + `确认拣货` → **拣货已完成**
  5. `执行发货` → **货物已发出** → `确认签收` → **货物已签收**
  6. `开具发票` → **发票已开具** → `生成应收账款` → **应收账款已生成**
  7. `登记收款并核销` → **应收账款已核销** (流程终态)
- **分支条件:** 折扣率 > 15% 或金额 > ¥500,000 → 价格审批 (R-35);信用超额 → 冻结 (R-01)。

### WF-02: 采购到付款工作流 (Procure-to-Pay)
- **触发事件:** **请购单已创建** (来源: **补货点已触达** 或 **缺货预警**)
- **编排策略:** 编制式编排 (Orchestration),由 SRM/ERP 主导调度。
- **参与角色:** 采购员、采购经理、供应链总监、收货员、质检员、仓管员、应付财务专员、财务经理。
- **动作序列:**
  1. `审批请购单` (分级 R-18) → **请购单已批准**
  2. `执行询价比价` (≥3 家 R-17/R-19) → **比价已完成**
  3. `创建采购订单` → **采购订单已下达**
  4. `登记收货` → **货物已到货** → `执行到货检验`
  5. 分支: 合格 → **质检已完成** → `执行入库` → **货物已入库** + **库存已增加**; 不合格 → **质检不合格** (转 WF-06)
  6. `生成应付账款` (三单匹配 R-25) → **应付账款已生成** (异常 → **三单匹配异常**,人工核查 R-26)
  7. `发起付款` → `审批付款` (分级 R-27) → `执行付款` → **付款已完成** (流程终态)
- **分支条件:** 预付供应商 → 先付款审批 (R-22);三单差异 > ¥1,000 → 人工放行 (R-26)。

### WF-03: 库存补货与缺货联动工作流
- **触发事件:** **缺货预警** 或 **补货点已触达**
- **编排策略:** 事件驱动,WMS 与 ERP/SRM 联动,体现"缺货转采购"闭环。
- **参与角色:** 库存计划员、补货引擎、采购员。
- **动作序列:**
  1. 接收 **缺货预警** / **补货点已触达**
  2. `创建请购单` (来源标记 缺货转采购 / 补货点自动) → **请购单已创建** (汇入 WF-02)
  3. WF-02 完成后发出 **库存已增加**
  4. `检查并占用库存` 自动重试,对 `欠交` 行按优先级队列 (R-07) 重新占用 → **库存已占用** (回归 WF-01 步骤 4)
- **分支条件:** 客户接受延期 → 缺货等待 (Backorder);否则缺货转采购 (R-05)。

### WF-04: 客户退货与应收红冲工作流 (Return-to-Refund)
- **触发事件:** 销售代表发起 `创建退货单` (须签收后 7 个工作日内,R-23)
- **编排策略:** 编制式编排,ERP 主导,WMS 与 FMS 协同。
- **参与角色:** 销售代表、销售经理、质检员、仓管员、应收财务专员。
- **动作序列:**
  1. `创建退货单` (退货类型=客户退货) → **退货单已创建**
  2. `验收退货` (质检员) → 合格品继续,不合格品报废
  3. `退货入库` → **退货已入库** + **库存已增加**
  4. 开具红字 `发票` → 对原 `应收账款` 红冲 → **应收账款已红冲**
- **分支条件:** 逾期申请 → 销售经理特批 (R-23);商品损坏 → 不入库直接报废。

### WF-05: 信用冻结审查工作流
- **触发事件:** **订单信用冻结**
- **编排策略:** 人工 + 系统混合,信用专员主导。
- **参与角色:** 信用专员、销售经理、财务经理。
- **动作序列:**
  1. 接收 **订单信用冻结**
  2. 信用专员审查: 客户预付 / 临时提额 (R-02/R-39)
  3. 分支: 满足条件 → `解除信用冻结` → **信用冻结已解除** (回归 WF-01 步骤 3)
  4. 分支: 5 个工作日未解除 → 订单自动 `已取消` → **订单已取消** + 释放占用库存 (R-03)
- **分支条件:** 临时提额超战略客户上限 ¥5,000,000 → 拒绝 (R-39)。

### WF-06: 退供应商与应付冲减工作流
- **触发事件:** **质检不合格**
- **编排策略:** 事件驱动,SRM 与 WMS、ERP 协同。
- **参与角色:** 采购员、仓管员、应付财务专员。
- **动作序列:**
  1. 接收 **质检不合格** → `创建退供单` (退货类型=退供应商,5 个工作日内 R-24) → **退供单已创建**
  2. `退货出库` (按不合格批次,不入正常库) → **退供已出库**
  3. 冲减对应 `应付账款` → **应付账款已冲减**
- **分支条件:** 供应商连续 3 批不合格 → 触发供应商评级下调 / 列入黑名单 (R-21)。

## 4. 编排原则与终态

- 所有工作流的跨系统步骤一律经由**事件总线**异步驱动 (R-40);同一事件可被多个消费者订阅 (如 **库存已增加** 同时供 WF-03 重试占用与 BI 库龄计算)。
- **库存权威值**以 WMS 为准;**财务往来权威值**以 ERP/FMS 为准 (见 `03-系统与数据实体.md` §4)。
- 终态事件: **应收账款已核销** (WF-01)、**付款已完成** (WF-02)、**应收账款已红冲** (WF-04)、**应付账款已冲减** (WF-06)、**坏账已计提** (逾期异常终态)。
