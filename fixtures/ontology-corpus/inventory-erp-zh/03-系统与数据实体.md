**文档编号:** ARC-ERP-INV-DATA-04

## 1. 目的与范围

本文件定义进销存全流程涉及的**记录系统 (System of Record)**、各系统的**核心数据实体及属性** (标注类型 / 主键 PK / 外键 FK / 枚举值)、**系统间集成关系** 以及**系统间交换的事件**。命名与 `01-业务流程SOP.md`、`02-业务规则与制度.md`、`04-事件与工作流编排.md` 全文一致。

---

## 2. 记录系统总览

| 系统 | 简称 | 职责 | 主数据归属 |
|---|---|---|---|
| ERP核心系统 (天衡ERP) | ERP | 销售订单、采购订单、发票、应收/应付主数据中枢、商品主数据 | `商品SKU`、`销售订单`、`采购订单`、`发票`、`应收账款`、`应付账款` |
| WMS仓储管理系统 | WMS | 库位、批次、库存事务、收货、入库、拣货、发货、盘点、调拨 | `库存`、`收货单`、`入库单`、`出库单`、`发货单`、`盘点单`、`调拨单` |
| CRM客户关系管理系统 | CRM | 客户主数据、报价、信用额度维护 | `客户`、`报价单`、`信用额度` |
| SRM供应商关系管理系统 | SRM | 供应商主数据、请购、询价比价 | `供应商`、`请购单` |
| FMS财务结算系统 | FMS | 收款、付款、对账、坏账、总账过账 | `收款单`、`付款单`、`对账单` |
| BI数据分析平台 | BI | 库龄、呆滞、周转率、信用风险分析 (只读派生) | (无主数据,消费各系统事件) |

## 3. 核心数据实体与属性

### 3.1 商品SKU (主系统: ERP)
- `skuId` — 字符串 — **PK**
- `商品名称` — 字符串
- `规格型号` — 字符串
- `计量单位` — 枚举: {个, 箱, 千克, 米, 套}
- `安全库存` — 整数
- `补货点` — 整数
- `经济订货量EOQ` — 整数
- `批次管理标志` — 布尔
- `保质期天数` — 整数 (0 表示无保质期)
- `ABC分类` — 枚举: {A, B, C}
- `默认库位` — 字符串 — **FK → 库存.库位编码**
- `商品状态` — 枚举: {正常, 停售, 待清理}

### 3.2 客户 (主系统: CRM)
- `客户编号` — 字符串 — **PK**
- `法定名称` — 字符串
- `税号` — 字符串
- `客户等级` — 枚举: {普通, 重要, 战略}
- `客户状态` — 枚举: {待审核, 正常, 冻结, 停用}
- `信用状态` — 枚举: {正常, 信用冻结, 黑名单}
- `结算方式` — 枚举: {预付, 货到付款, 账期}

### 3.3 信用额度 (主系统: CRM,镜像至 ERP)
- `信用额度ID` — 字符串 — **PK**
- `客户编号` — 字符串 — **FK → 客户.客户编号**
- `信用额度` — 金额
- `已用额度` — 金额 (= 未结应收 + 在途订单金额)
- `账期天数` — 整数 (默认 30)
- `生效日期` / `失效日期` — 日期

### 3.4 供应商 (主系统: SRM)
- `供应商编号` — 字符串 — **PK**
- `供应商名称` — 字符串
- `供应商状态` — 枚举: {合格, 待评估, 黑名单, 停用}
- `结算条件` — 枚举: {预付, 月结30, 月结60, 货到付款}
- `账期天数` — 整数
- `评级` — 枚举: {A, B, C, D}

### 3.5 报价单 (主系统: CRM)
- `报价单号` — 字符串 — **PK**
- `客户编号` — 字符串 — **FK → 客户**
- `报价金额` — 金额
- `折扣率` — 小数
- `报价状态` — 枚举: {待审批, 已批准, 已驳回, 已转单, 已过期}
- `有效期` — 日期

### 3.6 销售订单 (主系统: ERP)
- `销售订单号` — 字符串 — **PK**
- `客户编号` — 字符串 — **FK → 客户**
- `报价单号` — 字符串 — **FK → 报价单**
- `订单金额` — 金额
- `优先级` — 枚举: {普通, 加急, VIP}
- `订单状态` — 枚举: {已创建, 信用冻结, 已确认, 部分发货, 已发货, 已签收, 已完成, 已取消}
- `创建时间` — 时间戳

### 3.7 订单明细 (主系统: ERP,被销售订单与采购订单复用)
- `明细ID` — 字符串 — **PK**
- `单据号` — 字符串 — **FK → 销售订单 / 采购订单**
- `skuId` — 字符串 — **FK → 商品SKU**
- `数量` / `已发数量` / `已收数量` — 整数
- `单价` — 金额
- `行状态` — 枚举: {正常, 缺货, 欠交, 已完成}

### 3.8 请购单 (主系统: SRM)
- `请购单号` — 字符串 — **PK**
- `skuId` — 字符串 — **FK → 商品SKU**
- `请购数量` — 整数
- `请购来源` — 枚举: {补货点自动, 缺货转采购, 手工}
- `请购状态` — 枚举: {待审批, 已批准, 已驳回, 已转单}

### 3.9 采购订单 (主系统: ERP)
- `采购订单号` — 字符串 — **PK**
- `供应商编号` — 字符串 — **FK → 供应商**
- `请购单号` — 字符串 — **FK → 请购单**
- `采购金额` — 金额
- `结算条件` — 枚举: {预付, 月结30, 月结60, 货到付款}
- `采购状态` — 枚举: {已下达, 部分到货, 已到货, 已入库, 已关闭}

### 3.10 收货单 (主系统: WMS)
- `收货单号` — 字符串 — **PK**
- `采购订单号` — 字符串 — **FK → 采购订单**
- `到货数量` / `合格数量` / `不合格数量` — 整数
- `收货状态` — 枚举: {待质检, 质检中, 已质检, 已入库, 部分退货}

### 3.11 入库单 (主系统: WMS)
- `入库单号` — 字符串 — **PK**
- `收货单号` — 字符串 — **FK → 收货单**
- `批次号` — 字符串
- `入库数量` — 整数
- `入库日期` — 日期
- `库位编码` — 字符串 — **FK → 库存.库位编码**

### 3.12 库存 (主系统: WMS) —— 库存按 SKU × 库位 × 批次 维度记录
- `库存ID` — 字符串 — **PK** (= skuId + 库位编码 + 批次号)
- `skuId` — 字符串 — **FK → 商品SKU**
- `库位编码` — 字符串
- `批次号` — 字符串
- `现存量` — 整数 (不得为负,见 R-06)
- `占用量` — 整数
- `可用库存` — 整数 (派生 = 现存量 − 占用量)
- `入库日期` — 日期 (用于 FIFO / 库龄)
- `生产日期` / `到期日期` — 日期

### 3.13 出库单 (主系统: WMS)
- `出库单号` — 字符串 — **PK**
- `销售订单号` — 字符串 — **FK → 销售订单**
- `库存ID` — 字符串 — **FK → 库存**
- `出库数量` — 整数
- `出库状态` — 枚举: {待拣货, 已拣货, 已发货}

### 3.14 发货单 (主系统: WMS)
- `发货单号` — 字符串 — **PK**
- `出库单号` — 字符串 — **FK → 出库单**
- `承运商` — 字符串
- `运单号` — 字符串
- `发货状态` — 枚举: {已发货, 在途, 已签收}

### 3.15 退货单 (主系统: ERP,客户退货 / 退供应商共用)
- `退货单号` — 字符串 — **PK**
- `退货类型` — 枚举: {客户退货, 退供应商}
- `来源单号` — 字符串 — **FK → 发货单 / 采购订单**
- `退货数量` — 整数
- `退货状态` — 枚举: {待验收, 已验收, 已入库, 已出库}

### 3.16 发票 (主系统: ERP)
- `发票号` — 字符串 — **PK**
- `发票类型` — 枚举: {蓝字, 红字}
- `关联单号` — 字符串 — **FK → 发货单 / 收货单**
- `发票金额` — 金额
- `开票日期` — 日期
- `发票方向` — 枚举: {销项, 进项}

### 3.17 应收账款 (主系统: ERP)
- `应收单号` — 字符串 — **PK**
- `客户编号` — 字符串 — **FK → 客户**
- `发票号` — 字符串 — **FK → 发票**
- `应收金额` / `已收金额` — 金额
- `到期日` — 日期 (= 开票日 + 账期天数)
- `应收状态` — 枚举: {未结, 部分收款, 已结清, 逾期, 坏账}

### 3.18 应付账款 (主系统: ERP)
- `应付单号` — 字符串 — **PK**
- `供应商编号` — 字符串 — **FK → 供应商**
- `采购订单号` — 字符串 — **FK → 采购订单**
- `应付金额` / `已付金额` — 金额
- `到期日` — 日期 (= 对账确认日 + 账期天数)
- `应付状态` — 枚举: {未付, 部分付款, 已付清, 已逾期}

### 3.19 收款单 (主系统: FMS)
- `收款单号` — 字符串 — **PK**
- `客户编号` — 字符串 — **FK → 客户**
- `收款金额` — 金额
- `核销发票号` — 字符串 — **FK → 发票**
- `收款方式` — 枚举: {银行转账, 票据, 现金}

### 3.20 付款单 (主系统: FMS)
- `付款单号` — 字符串 — **PK**
- `供应商编号` — 字符串 — **FK → 供应商**
- `付款金额` — 金额
- `核销应付单号` — 字符串 — **FK → 应付账款**
- `付款状态` — 枚举: {待审批, 已审批, 已付款}

### 3.21 对账单 (主系统: FMS)
- `对账单号` — 字符串 — **PK**
- `对账对象类型` — 枚举: {客户, 供应商}
- `对账对象编号` — 字符串 — **FK → 客户 / 供应商**
- `账期区间` — 日期范围
- `对账状态` — 枚举: {待确认, 已确认, 有差异}

### 3.22 盘点单 (主系统: WMS)
- `盘点单号` — 字符串 — **PK**
- `库位范围` — 字符串
- `账面量` / `实盘量` / `差异量` — 整数
- `差异率` — 小数
- `盘点状态` — 枚举: {进行中, 待复核, 已调账}

### 3.23 调拨单 (主系统: WMS)
- `调拨单号` — 字符串 — **PK**
- `出库仓` / `入库仓` — 字符串
- `skuId` — 字符串 — **FK → 商品SKU**
- `批次号` — 字符串
- `调拨数量` — 整数
- `调拨状态` — 枚举: {待出库, 在途, 已入库}

### 3.24 价格 (主系统: ERP)
- `价格ID` — 字符串 — **PK**
- `skuId` — 字符串 — **FK → 商品SKU**
- `价目表` — 枚举: {普通价, 重要客户价, 战略客户价, 采购价}
- `单价` — 金额
- `生效日期` / `失效日期` — 日期

## 4. 系统间集成与事件交换

| 上游系统 → 下游系统 | 集成方式 | 交换的事件 / 数据 |
|---|---|---|
| CRM → ERP | 主数据同步 (实时) | **客户已创建**、**信用额度已核定**、**报价单已批准** |
| ERP → WMS | 事件总线 (异步) | **信用校验通过** (触发占用)、**采购订单已下达** (预期收货)、`商品已创建` 主数据 |
| WMS → ERP | 事件总线 (异步) | **库存已占用**、**货物已签收**、**货物已入库**、**库存已增加**、**缺货预警** |
| SRM → ERP | API 调用 | **请购单已批准**、**比价已完成** (转采购订单) |
| ERP → FMS | 事件总线 (异步) | **发票已开具**、**应收账款已生成**、**应付账款已生成** |
| FMS → ERP | 回写 (异步) | **收款已登记**、**付款已完成** (核销状态回写) |
| WMS / ERP / FMS → BI | 事件流订阅 (近实时) | 全部库存事务、应收应付账龄、对账差异 (用于库龄、呆滞、风险分析) |

**集成原则:** 所有跨系统的单据状态变更必须通过**事件总线**异步同步 (规则 R-40),下游系统的派生状态不得人工直接修改。库存的权威值始终以 **WMS** 为准,财务往来的权威值以 **ERP/FMS** 为准。
