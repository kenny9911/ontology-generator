## 写在前面（请新人务必先看完这一段）

这份手册是给三类人看的：总部和职能后勤的**内勤员工**、研发中心的**研发员工**、以及驻场或项目制的**外包员工**。这三类人虽然都在鼎财云里走报销，但是能报什么、报多少、找谁签字、要交什么单据，差别很大。很多新来的同事、新进场的外包，一上来就拿着内勤的标准去报研发设备，或者拿着正式员工的标准去报外包的餐补，结果单子被退回来反复折腾，既耽误自己拿钱，也让我们审核科加班。所以请先花二十分钟把第一部分关于你这一类人的规则看透，再看第二部分跟你相关的特殊场景。

先把几个全集团统一的概念说清楚，后面就不重复了：

**两条审批线是并行又串联的。** 一张报销单提交后，先走**业务部门审批**这条线：经办人提交 → 直属上级 → 部门负责人 → 事业部分管领导，具体走到哪一级，看金额。业务线签完，单子才会流到**财务审批**这条线：费用会计初审 → 财务复核 → 财务经理/财务总监终审，同样看金额和科目升级。两条线都通过，钱才会打款。所以你看到单子"在财务"不代表马上到账，财务这边自己还有三级。

**金额阈值（这是全集团的基础线，三类员工在此基础上各有差异，后面分别讲）：**

- 单笔/单张报销单 ≤ 2,000 元：直属上级审批即可（业务线），费用会计初审通过即过（财务线）。
- 2,000 元 < 金额 ≤ 10,000 元：业务线走到部门负责人；财务线走到财务复核。
- 10,000 元 < 金额 ≤ 50,000 元：业务线走到事业部分管领导；财务线走到财务经理。
- 金额 > 50,000 元：业务线分管领导签完，财务线必须财务总监终审；超过 200,000 元的，财务总监之外还要分管财务的集团副总裁会签。

**几个常用人物（后面案例会反复出现，先混个脸熟）：**

- 集团财务总监：周敏总。50,000 元以上的单子最终都到她这。
- 费控科科长：郑国强。制度归口、阈值解释权在费控科。
- 报销审核科科长：李文娟。日常审单、退单、咨询找她这条线。
- 外包管理办公室主任：黄涛。外包人员的进场、对接人备案、对私打款审批，归他这块。
- 集团差旅平台对接人：行政中心 孙立。机票酒店、差旅标准的事找他。

**一句话原则：** 不该你报的别报，能走对公的别走对私，能事前申请的别先斩后奏。这三句话记住，你的报销 90% 都不会被退。

# 第一部分：内勤 / 研发 / 外包三类员工差异化

## 1. 三类员工总览（先看这张对照表）

下面这张表是全手册的"地图"，先看个大概，细节在后面 2、3、4 节分别展开。

| 维度 | 内勤员工 | 研发员工 | 外包员工 |
|---|---|---|---|
| 身份 | 集团正式编制，总部及各职能后勤 | 集团正式编制，挂研发中心 | 第三方派遣/项目制，签的是供应商合同 |
| 鼎财云账号 | 正式工号 + 鼎财账号 | 正式工号 + 鼎财账号 + 研发专项标签 | 外包工号（W 开头）+ 受限报销权限 |
| 可报范围 | 办公、差旅、招待、福利等全科目 | 在内勤基础上 + 研发专项（设备、试剂耗材、样机、专利、外部测试等） | 仅限项目明确约定的范围，默认只有交通、加班餐、必要耗材 |
| 招待费 | 可报（有额度，需事前报备大额招待） | 可报，但研发对外招待较少 | 一律不可报招待费 |
| 福利/团建 | 享受集团及部门福利费 | 同内勤 | 不享受集团福利费，团建是否参与由项目和供应商约定 |
| 业务审批起点 | 直属上级 | 直属上级（研发专项加签研发项目负责人） | 经办人 → 甲方项目经理/对接人 → 外包供应商对接 → 集团对接部门负责人 |
| 打款方式 | 对公优先，个人垫付走对私报销 | 同内勤 | 默认通过供应商结算，个人对私打款受严格限制（见 4.5） |
| 差旅标准 | 按职级套用差旅标准表 | 同职级内勤标准，研发外出测试有专项 | 按项目合同约定，通常对标内勤普通职级或更低，且需甲方确认 |
| 月度报销时限 | 次月 10 日前报上月费用 | 同内勤 | 同内勤，但需先过甲方对接人确认，建议提前 3 个工作日 |

记住这张表的核心差异点：**内勤是"标准全集"，研发是"标准全集 + 研发专项"，外包是"受限子集 + 甲方确认 + 对私严控"。**

## 2. 内勤员工报销规则

内勤员工指的是总部职能部门和后勤支持的正式员工，比如行政、人力、财务、法务、采购、IT 支持、综合管理这些岗位。内勤是最"标准"的一类，研发和外包都是在内勤的基础上做加减法，所以内勤这一节写得最细，后面两类会引用这里。

### 2.1 内勤可报科目与限额

**（1）日常办公费用**

- 办公用品（笔、本、文件夹、打印纸等）：单次 ≤ 500 元可由直属上级签批；超过 500 元的批量采购建议走采购部统采，不要个人垫付。
- 办公设备小件（键盘、鼠标、U 盘、移动硬盘、网线等）：单件 ≤ 800 元可报，超过的走 IT 资产申请流程（鼎财里选"资产类申请"而不是"费用报销"）。
- 名片印制：每人每次 ≤ 200 元，原则上一年最多两次。
- 部门内部小额茶水、纸巾、绿植：挂"行政杂费"，单月单部门累计建议 ≤ 1,000 元，超出需部门负责人说明。

**（2）市内交通费**

- 因公打车（滴滴企业版优先，自动对公）：单次 ≤ 100 元正常报，超过 100 元需在备注里写清起止地点和事由。
- 个人垫付的市内交通（地铁、公交、偶发打车）：凭票或行程单报销，月度累计 ≤ 800 元，超出部分需直属上级特批。
- 提醒：能用滴滴企业版的尽量用企业版，对公直接结算，不用你垫钱，也不用贴票，省心。

**（3）通讯补贴**

- 内勤通讯补贴按职级随工资发放，**不再单独凭票报销话费**。如果你在鼎财里提交话费发票，会被费用会计直接退单，理由写"通讯已随薪发放，不重复报销"。

**（4）招待费**

- 工作餐招待（陪同来访客户、兄弟单位）：人均标准——总部所在城市 ≤ 200 元/人，外地 ≤ 150 元/人。
- 单次招待总额 ≤ 2,000 元，直属上级签批；> 2,000 元需部门负责人签批；> 5,000 元的招待**必须事前在鼎财提"招待事前报备单"**（见第二部分场景 8）。
- 招待报销单必须写清：被招待单位、人数、姓名（至少主要陪同人）、事由。只写"招待客户"四个字的，一律退单。
- 酒水：单瓶白酒 ≤ 500 元，整单酒水金额不得超过餐费的 50%。超的部分自付。

**（5）会议费**

- 部门内部小型会议茶歇、矿泉水：单次 ≤ 800 元。
- 对外或跨部门大型会议（含场地、设备、餐饮）：需事前提"会议费预算申请单"，按金额阈值审批。

**（6）培训费**

- 个人因岗位需要参加的外部培训：需事前经直属上级和人力审批，凭培训机构发票报销。单次 ≤ 5,000 元部门负责人批，超出走人力的年度培训预算。

### 2.2 内勤差旅标准

差旅这块全集团有一张《鼎垣差旅标准表》，内勤按职级套用。这里把内勤常用的几档列出来（金额为上限，节俭从简）：

| 职级 | 住宿（元/晚，按城市分档） | 市内交通（元/天） | 伙食补助（元/天） |
|---|---|---|---|
| 普通职员（P1-P3） | 一线城市 450 / 其他 350 | 80 | 100 |
| 主管/经理（P4-P5/M1） | 一线城市 600 / 其他 450 | 100 | 120 |
| 部门负责人（M2-M3） | 一线城市 800 / 其他 600 | 150 | 150 |
| 事业部分管及以上 | 据实从严，提前报备 | 据实 | 据实 |

- **机票：** 普通职员经济舱；主管/经理经济舱（紧急情况可订全价经济舱）；总监级以上可公务舱（4 小时以上航程）。机票一律走集团差旅平台（找孙立的那个平台），不要自己在外面订了再来报，自己订的除非平台当时确实订不到，否则报销审核科有权按平台同档价封顶。
- **高铁：** 普通职员二等座，经理及以上一等座，总监级商务座（视情况）。
- **住宿超标：** 因房源紧张确实订不到标准内房间的，留好截图，在备注说明，部门负责人签字可放行；无说明的超标部分自付。
- **伙食补助：** 是包干的，发了补助就不能再单独报餐费发票。出差期间公司已安排用餐（如会议供餐）的，当餐补助扣减。

### 2.3 内勤审批路径（具体到人）

以一张内勤的差旅报销单为例，金额 8,600 元：

1. 经办人在鼎财提交，附行程单、发票、出差申请单编号；
2. 业务线：直属上级（比如行政科长）→ 因为超过 2,000 元，继续到部门负责人（行政中心总监）；
3. 财务线：费用会计初审（查发票真伪、查票通、查标准）→ 因为超过 2,000 元，到财务复核；
4. 复核通过，因为没超 10,000 元，财务经理这一级不需要，直接进入打款排程。

打款一般是 T+3 到 T+5 个工作日，节点是每周二、周五两批集中付款。

### 2.4 内勤单据要求

- 发票必须是增值税发票（普票或专票），抬头一律"鼎垣智造集团有限公司"，税号见鼎财首页公告，**别写成简称"鼎垣"**，简称的发票票通查得到但不合规，会退。
- 电子发票必须在票通能查到且未被重复报销（鼎财会自动比对发票号查重）。
- 餐饮、住宿等需配行程或事由说明。
- 大额（≥ 5,000 元）尽量取得专票，便于抵扣。

## 3. 研发员工报销规则

研发员工指挂在各研发中心、研究院、技术预研部门的正式员工。研发员工的日常费用（办公、差旅、市内交通、招待）**完全套用上面第 2 节内勤的标准，不再重复**。这一节只讲研发独有的、内勤没有的部分——也就是"研发专项"。

### 3.1 研发专项的核心特征

研发专项费用在鼎财里有个独立的"研发专项"标签，提交时必须**关联具体研发项目编号**（比如 RD-2024-0137 这种）。没有项目编号的研发费用，系统不让提交，因为这些费用最后要归集到研发项目里做研发费用加计扣除（这关系到集团税务，所以卡得很死）。

研发专项的业务审批，在普通审批线之外，**必须加签该研发项目的项目负责人（PI）**。也就是说研发员工报设备，链路是：经办人 → 直属上级 → **研发项目负责人** → 研发中心负责人 →（按金额）分管领导。项目负责人这一签是确认"这笔确实是我项目要花的、预算里有"。

### 3.2 研发专项可报科目与限额

**（1）研发设备/仪器**

- 单件 < 5,000 元的小型仪器、工具、测量设备：可走费用报销（计入低值易耗或研发费用），需研发项目负责人签字。
- 单件 ≥ 5,000 元的设备：**不走费用报销，走固定资产采购流程**。在金蝶云里建资产卡片，需提前做"设备采购申请"，按集团采购制度可能要走比价或招投标（见第二部分场景 8）。研发员工最常踩的坑就是自己先买了一台一万多的示波器再来报，这是不行的，固定资产必须事前申请、入资产账。
- 设备配件、备件、维修：据实报销，需说明对应设备和项目。

**（2）试剂、耗材、样品**

- 实验试剂、耗材：据实报销，需附清单（什么试剂、多少量、单价），关联项目。单张 ≤ 20,000 元研发中心负责人签批，超出走分管领导。
- 易制毒、易制爆等管制类试剂：除报销外还需附公安备案/采购资质记录，缺一不可，否则退单且通报 EHS。
- 外购样品、竞品（用于对标拆解分析）：需说明用途，关联项目，单件 ≤ 10,000 元正常报，超出报备。

**（3）样机制作 / 打样 / 外协加工**

- 找外部供应商打样、做样机、外协加工：**金额通常较大，优先走对公采购合同**，不要个人垫付几万块再来报。确需小额垫付打样的，单笔 ≤ 3,000 元可个人报销，超出请走采购对公付款。

**（4）外部测试 / 检测 / 认证费**

- 送第三方机构做检测、认证、可靠性测试：凭检测报告 + 发票报销，关联项目。这类一般金额较大且对公开票，强烈建议走对公付款而非个人垫付。

**（5）专利 / 知识产权费**

- 专利申请代理费、官费、年费、商标注册等：走法务/知识产权归口流程，研发员工经办的需附受理通知书或代理所发票，关联项目。

**（6）研发差旅专项**

- 研发员工因项目去外部实验室、客户现场做联调测试、参加技术展会/学术会议的差旅：套用内勤同职级差旅标准，但伙食补助在现场连续测试超过 12 小时/天的，可申请加班餐补（见下条），需项目负责人确认。

**（7）加班餐补（研发高频）**

- 研发员工因项目赶进度加班，加班至 20:00 之后的，可报当晚加班餐，标准 ≤ 50 元/人/次，凭票或食堂记录。整月加班餐由项目负责人统一确认核实，不能虚报。注意：内勤员工加班餐标准同为 ≤ 50 元/人/次，但需直属上级确认。

### 3.3 研发专项审批阈值差异

研发专项因为涉及加计扣除和项目预算，阈值比内勤的通用阈值**更严一档**：

| 研发专项金额 | 业务线终点 | 财务线终点 |
|---|---|---|
| ≤ 5,000 元 | 项目负责人 + 研发中心负责人 | 费用会计初审 |
| 5,000 - 20,000 元 | + 研发中心负责人复核 | 财务复核 |
| 20,000 - 50,000 元 | + 分管研发领导 | 财务经理 |
| > 50,000 元 | + 分管研发领导 + CTO 知会 | 财务总监周敏 |

注意：研发设备一旦 ≥ 5,000 元就转固定资产流程，所以上表"费用报销"口径主要针对耗材、试剂、测试费、小额工具等。

### 3.4 研发单据特别要求

- 所有研发专项报销，**项目编号是强制字段**，没有就提交不了。
- 试剂耗材类必须有明细清单，不能只有一张总额发票。
- 检测/认证类必须附检测报告或认证证书编号。
- 研发费用归集对税务很关键，费用会计会重点核对"费用性质与项目是否匹配"，比如你拿一个市场推广的费用挂研发项目，会被打回。

## 4. 外包员工报销规则（重点中的重点，请逐字看）

外包员工是三类里规则最特殊、最容易出问题的。请所有外包同事、以及管理外包的集团对接人，把这一节看完整。

### 4.1 外包员工的身份与基本原则

外包员工不是集团正式编制，签的是与第三方供应商（人力派遣公司或项目外包公司）的合同，在鼎财里是 **W 开头的外包工号**，报销权限是**受限的**。基本原则四条：

1. **可报范围以项目合同约定为准，没约定的默认不可报。** 外包员工不像内勤"标准全集"，外包是"白名单"——只有合同/进场约定里明确写了可报的，才能报。默认白名单只有三项：因公交通、加班餐、项目必需的小额耗材。
2. **任何报销都需要甲方（也就是我们集团这边）项目经理或对接人确认。** 外包提的单子，业务审批第一道不是供应商，而是**集团这边的项目经理/对接人**先确认"这事确实是我让他干的、费用我认"。
3. **外包不享受招待费、福利费、团建费（除非项目另有约定）。** 外包同事经常误报这些，一律退。
4. **对私打款严格受限。** 能通过供应商月度结算的，一律走供应商；个人对私打款是例外，要走专门审批（见 4.5）。

### 4.2 外包可报范围（白名单）与限额

**（1）因公交通费**

- 因项目需要在城市内的因公打车、公共交通：凭票/行程单报销，**月度累计 ≤ 600 元**（比内勤的 800 元低）。超出需甲方项目经理书面确认 + 外包供应商对接同意。
- 跨城市出差：外包出差必须**事前**由甲方项目经理在鼎财发起"外包出差确认"，未经事前确认的外包出差费用不予报销。差旅标准对标内勤普通职员（P1-P3）档，且住宿、交通从严。

**（2）加班餐费**

- 因项目加班至 20:00 之后：≤ 50 元/人/次，凭票，需甲方对接人确认加班事实。
- 提醒：外包加班餐报销必须有甲方对接人在加班记录或鼎财备注里确认，不能只凭供应商一方说法。

**（3）项目必需的小额耗材**

- 项目执行直接需要的小额耗材、工具：**单笔 ≤ 1,000 元**，需甲方项目经理确认确实是项目需要。超过 1,000 元的，一律走集团采购对公购买，不允许外包个人垫付。

**（4）明确不可报（外包黑名单，常见误报，重点提醒）**

- 招待费、礼品、酒水——一律不可报。
- 福利费、节日费、团建费——不可报（除非合同明确约定，且由外包管理办公室黄涛主任确认）。
- 通讯费、话费——不可报。
- 培训费、考证费——属于供应商对自己员工的责任，不可向甲方报。
- 大额采购、设备——不可报，必须走集团对公采购。
- 个人交通通勤（上下班）——不可报，因公才行。

### 4.3 外包审批路径（和内勤完全不同，看清楚）

外包的业务审批链路有四道，缺一不可：

1. **经办人（外包本人）** 在鼎财以 W 工号提交，必须填写所属项目、甲方对接人；
2. **甲方项目经理 / 对接人确认** ——这是核心一道，确认费用真实、与项目相关、金额合理；
3. **外包供应商对接人**（供应商驻场负责人或供应商财务）——确认该笔费用属于合同约定可报范围、不与服务费重复；
4. **集团对接部门负责人** ——按金额，超过一定额度（外包阈值见下）再往上。

然后才进财务线（费用会计 → 复核 → 按额经理/总监）。

**外包金额阈值（比内勤更低、更严）：**

| 外包报销金额 | 业务线终点 | 备注 |
|---|---|---|
| ≤ 500 元 | 甲方对接人 + 供应商对接 | 标准白名单内 |
| 500 - 2,000 元 | + 集团对接部门负责人 | 需事由说明 |
| 2,000 - 5,000 元 | + 事业部分管领导 + 外包管理办黄涛知会 | 一般是出差类 |
| > 5,000 元 | 原则上不应出现个人报销，应走供应商结算 | 例外需财务总监周敏特批 |

### 4.4 代垫与结算（外包费用怎么走账）

外包费用有两种走账方式，请分清：

**方式 A：供应商月度结算（默认、首选）**
项目期间外包发生的可报费用，由外包本人在鼎财提交、甲方对接人确认后，**汇总进供应商当月结算单**，集团对公付给供应商，供应商再发给员工个人。这是最规范的方式，因为它和供应商的服务费走在一条对公链路上，账目清楚、不涉及向外部个人对私打款的税务和合规风险。

- 操作上：外包当月费用累计 → 月底由甲方对接人确认明细 → 进供应商对账单 → 供应商开具增值税发票（费用部分通常并入服务费或单列"代垫费用"）→ 集团对公付款。

**方式 B：员工个人对私报销（例外、严控，见 4.5）**
只有在合同明确约定、或经特批的情况下，才允许直接打到外包个人的私人账户。

### 4.5 外包对私打款规则（红线，请所有人记牢）

向外部个人（非集团员工）的私人账户打款，涉及税务、反洗钱、合规多重风险，所以集团对外包对私打款卡得非常死。规则如下：

1. **能走供应商结算的，绝对不允许走对私。** 这是第一条铁律。费用会计看到外包的对私报销单，第一反应就是问"为什么不进供应商结算？"
2. **确需对私的，必须满足以下全部条件：**
   - 合同里明确约定了某类费用可由甲方直接报销给外包个人；**或**有外包管理办公室黄涛主任的书面特批；
   - 甲方项目经理确认 + 供应商对接人书面同意（供应商需确认"该笔不再向我方重复报销"，避免双重支付）；
   - 单笔金额原则上 ≤ 2,000 元，**单人单月对私累计 ≤ 3,000 元**；
   - 提供外包本人身份证复印件、本人银行卡信息（卡号、开户行、户名必须与身份证一致）；
   - 财务复核 + 财务经理（≥ 2,000 元到财务总监）审批。
3. **对私打款一律不得用于规避供应商发票义务。** 也就是说，不能因为供应商不愿意开发票，就让外包个人来对私报销绕过去。这种发现一起处理一起。
4. **离场外包的对私清算**：外包离场前必须结清，离场后原则上不再受理对私报销（见第二部分场景 11）。

**案例（真实改编）：** 某事业部一外包驻场工程师，项目期间垫付了一台投影仪 4,200 元，想走对私报销。报销审核科李文娟退单，理由：(1) 投影仪 4,200 元超过外包耗材 1,000 元上限，属于资产采购，应走集团对公；(2) 即便允许，4,200 元也超过对私单笔 2,000 元上限。最终处理：投影仪由集团采购补开对公采购单，对公付给供应商，外包个人拿回垫付款。教训：外包不要垫付大额、不要垫付资产类，让甲方项目经理走对公采购才是正路。

### 4.6 外包单据要求

- 发票抬头同样是"鼎垣智造集团有限公司"全称（除非走供应商结算时由供应商代收，则开给供应商）。
- 走供应商结算的，最终是供应商对集团开票，外包个人留好原始凭证给供应商。
- 走对私报销的，原始发票必须给到集团财务、且票通可查、且未在供应商处重复报销。
- 所有外包报销必须能对应到具体项目和甲方对接人，"无主"的外包费用一律不收。

# 第二部分：特殊场景处理

下面每个场景都按"规则 → 阈值 → 单据 → 案例"来写，三类员工有差异的地方会专门标注。请按你遇到的场景对号入座。

## 场景 1：国内长差（连续出差超过一定天数）

**规则：**
- 出差前必须在鼎财提"出差申请单"，写明目的地、起止日期、事由、预计费用。先有申请单编号，回来才能报对应差旅。
- **连续出差 ≥ 15 天**算长差，长差期间住宿建议改为月租公寓或协议长住酒店，单价应低于短住标准，由行政差旅孙立协助协调。
- 长差伙食补助照常发放，但如果项目现场或客户已包餐，对应餐次补助扣减。
- 长差期间往返家中的探亲交通：连续出差满 **30 天**可报销一次往返经济舱机票或二等座高铁（普通职员），需部门负责人批。

**阈值：** 长差总费用往往超 10,000 元，业务线到分管领导，财务线到财务经理。

**单据：** 出差申请单编号、住宿发票（长住建议取整月发票）、交通票据、伙食补助按天计提（不另附餐票）。

**三类差异：**
- 内勤/研发：按职级标准，研发若是去现场做长期联调可叠加加班餐补（项目负责人确认）。
- 外包：长差必须甲方项目经理事前确认，住宿对标普通职员档且从严，探亲交通原则上不予报销（外包通勤/探亲属个人事项）。

**案例：** 研发员工小张去苏州客户现场做产线联调，连续 22 天。他出差前提了申请单（编号 BX-CC-2024-0451），现场住协议酒店每晚 380 元（在 P3 苏州 350 标准上略超，因房源紧张附了截图说明，研发中心负责人签字放行）。期间晚上连续联调常到 22 点，加班餐按 50 元/次报了 11 次，由项目负责人确认。第 22 天回来，伙食补助按 22 天×100=2,200 元计提，但其中 6 天客户管晚饭，扣减 6 餐补助。整单 13,400 元，走到分管研发领导 + 财务经理。

## 场景 2：海外出差

**规则：**
- 海外出差**必须事前**提"因公出国（境）申请单"，除了业务审批，还需经人力行政中心和（涉及保密/敏感地区时）合规法务会签。审批周期长，请至少提前 **10 个工作日**提交。
- 机票走集团差旅平台国际版，4 小时以上航程总监级以上可公务舱，其余经济舱。
- 海外住宿、伙食按《鼎垣海外差旅标准表》分国别/城市档，一般以美元或当地货币计标准，这里不一一列，找孙立要最新版。
- 海外伙食多为包干补助（按目的地分档，如发达国家城市约 80 美元/天，发展中国家约 50 美元/天，以当时标准表为准）。
- 签证费、国际旅行保险、必要的接种/检疫费可据实报销。

**汇率与外币（重点，单列见场景 9，这里先说核心）：**
- 海外发生的外币费用，**统一按回国报销当日中国银行现钞买入价**或财务指定的月度汇率折算成人民币入账，以财务公布口径为准，不允许自己挑一个对自己有利的汇率。
- 刷公司因公卡（如有）的外币消费，以银行实际入账人民币金额为准，最准确，优先用因公卡。

**阈值：** 海外出差金额普遍 > 10,000 元甚至 > 50,000 元，财务线常到财务经理/财务总监。

**单据：** 出国申请单、机票行程单、海外住宿发票/收据（外文收据需翻译要点并备注）、签证及保险票据、外币费用按场景 9 折算并附汇率截图。

**三类差异：**
- 外包员工原则上**不安排海外出差**；确有项目需要（如随项目赴海外现场）的，必须合同明确约定、外包管理办黄涛主任 + 财务总监周敏特批，标准从严，且优先走供应商结算而非对私。

**踩坑提醒：** 海外很多小票是手写收据或没有正规发票，回来贴票时务必在每张收据旁手写中文说明（日期、地点、事项、外币金额、折人民币），否则费用会计无法判断，会大面积退单。

## 场景 3：大额采购的事前预申请与招投标

**规则：**
- 单项采购（设备、服务、工程、大宗物料）金额达到下列阈值的，**必须事前**走采购流程，不允许个人先买后报：
  - 单项 ≥ 5,000 元：进入固定资产或集中采购范畴，需"采购申请单"。
  - 单项 ≥ 50,000 元：需 **3 家比价**（询比价），采购部归口。
  - 单项 ≥ 200,000 元：需走**正式招投标**或集团采购委员会评审。
  - 单项 ≥ 1,000,000 元：集团采购委员会 + 分管副总裁 + 财务总监会签，重大项目报集团总裁办。
- 报销环节：大额采购最终都应是**对公付款 + 对公发票/合同**，报销系统里只处理小额垫付，大额一律走采购付款流程（金蝶云里走采购订单 + 付款申请）。

**单据：** 采购申请单、比价记录/招标文件、采购合同、验收单、对公发票。缺验收单的，财务不予付尾款。

**三类差异：**
- 研发设备 ≥ 5,000 元强制走资产采购（见 3.2），这是研发最高频的大额场景。
- 内勤涉及的大额多为办公设备、行政服务采购，走采购部统采。
- 外包**不得**作为大额采购的经办垫付人，需求由外包提出、但采购主体必须是集团。

**案例：** 研发中心要买一台 18 万元的环境试验箱。流程：研发提需求 → 采购部组织 3 家以上比价（实际找了 4 家）→ 因接近 20 万门槛，走了简易招投标 → 签对公采购合同 → 到货验收 → 供应商开专票 → 金蝶云走付款申请 → 财务总监周敏审批付款。整个过程没有任何"报销"，研发员工一分钱都不用垫。如果当初有人图省事自己刷卡买了，那就是大错——18 万既无法走个人报销，又破坏了招投标合规，必然被通报。

## 场景 4：紧急采购 / 先斩后奏的补审

**规则：**
- 原则上**不鼓励先斩后奏**。但生产/研发/项目现场确有突发紧急情况（如设备突然故障停产、客户现场急需替换件），来不及走事前流程的，允许"先采购、后补审"，但必须满足：
  - 事中或事后 **24 小时内**电话/微信/邮件知会直属上级及对接的采购/财务，留痕；
  - 事后 **3 个工作日内**在鼎财补提"紧急采购补审单"，说明紧急原因、为何无法事前申请、采购明细、供应商；
  - 由部门负责人（金额大的到分管领导）确认紧急情况属实并签字。
- **没有合理紧急理由的"先斩后奏"，财务有权拒付，由经办人自行承担。** 这一条非常硬，别拿"我忘了申请"当紧急理由。

**阈值：**
- 紧急采购补审金额 ≤ 5,000 元：部门负责人确认即可。
- 5,000 - 50,000 元：分管领导确认 + 财务经理审核紧急合理性。
- > 50,000 元：原则上再紧急也要尽量走简易比价（哪怕事后补），财务总监周敏审批，且会复盘为什么会出现这种大额紧急。

**单据：** 紧急采购补审单、留痕的知会记录（截图）、采购发票、对应的故障/事件说明。

**三类差异：**
- 外包遇到现场紧急，**不要自己垫大额**，应立即电话甲方项目经理，由甲方决定走对公紧急采购。外包先斩后奏垫付大额的，大概率拿不回来，因为既无事前授权也超外包报销上限。

**案例：** 某事业部产线一台关键传感器深夜烧毁，停产待料。值班工程师（内勤研发协同岗）当晚电话报备直属上级和采购，第二天一早从本地供应商现购替换件 3,800 元先复产。次日在鼎财补提紧急采购补审单，附了停产工单照片和当晚的微信报备截图，部门负责人确认属实，顺利报销。反面例子：另一位同事月底想起来一笔忘了申请的 6,000 元办公家具采购，硬套"紧急"提交，被财务退回——家具采购不存在紧急性，属于该走没走的事前流程，最终走了说明 + 分管领导补签才勉强处理，且被点名提醒。

## 场景 5：对私 vs 对公报销

**规则（一句话：能对公就别对私）：**
- **对公优先级最高的场景：** 大额采购、长期服务、与供应商有合同的支出、可取得对公发票的支出，统统走对公付款（金蝶云付款申请），不要个人垫付。
- **对私（个人垫付报销）适用于：** 小额、零星、确实无法对公预付的因公支出（如临时打车、出差零星餐补、小额耗材）。
- 个人垫付报销打到本人工资卡（鼎财里绑定的本人账户），不允许打到他人账户。

**阈值参考（个人垫付的"软上限"，超过建议改对公）：**
- 内勤/研发个人单笔垫付 > 10,000 元的，财务会主动建议改对公，并核查为什么没走对公。
- 外包对私见 4.5，单笔 ≤ 2,000、单月 ≤ 3,000。

**单据：** 对公——合同 + 对公发票 + 付款申请；对私——本人垫付的合规发票 + 事由。

**踩坑提醒：** 最常见的错误是为了"省事"把本该对公的大额拆成几张个人报销单分次报，这叫**化整为零**，是明确违规的，鼎财会按同一供应商、同一事项、短期内多笔做关联预警，财务一旦发现，全部退回并要求改对公，情节重的通报。

## 场景 6：预付款与后续冲销核销

**规则：**
- 因公需要先付一笔钱（如出差备用金、活动预付定金、供应商预付款），走"借款/预付申请"，不是报销。
- **个人借款（备用金）：** 出差或活动前可借备用金，借款额度一般不超过预计费用的 80%，单人在途借款累计上限：普通职员 ≤ 10,000 元，经理级 ≤ 20,000 元，更高职级据实报备。
- **核销时限：** 出差/活动结束后 **15 个工作日内**必须用实际报销冲销借款，多退少补。
- **逾期不核销的后果：** 超过 15 个工作日未核销，鼎财自动提醒；超过 **30 天**仍未核销的，财务有权从工资中代扣借款，并暂停该员工新的借款权限。
- **对公预付款：** 供应商预付（如预付定金）走采购付款流程，凭合同付，到货/完工后凭发票和验收核销，不允许长期挂账。

**阈值：** 备用金借款 > 20,000 元需财务经理审批，对公预付 > 50,000 元需财务总监审批。

**单据：** 借款申请单 → 核销时附实际报销单据 → 系统自动计算多退少补。

**三类差异：**
- 外包员工**原则上不发放个人备用金**，项目所需预付通过供应商或集团对公解决。
- 研发项目大额预付（如外协打样定金）走对公采购预付，不走个人借款。

**案例：** 内勤员工小李借了 8,000 元备用金去北京参加 5 天展会。回来实际花了 6,800 元（住宿 2,800、交通 1,500、伙食补助 5×100=500、展会杂费 2,000）。在鼎财提报销单冲销借款，系统自动算出应退回公司 1,200 元，小李通过对公转账退回，借款核销完成，全程 9 个工作日，没超时。如果他拖到 31 天还没核销，财务就会从他工资里扣 8,000 元，他还得再申请退回多扣的部分，非常麻烦。

## 场景 7：发票丢失 / 不合规的处理

**规则：**
- **电子发票丢失：** 不算事，电子发票可在票通或开票方处重新下载，重新上传即可。所以现在能开电子发票的尽量开电子的。
- **纸质发票丢失：**
  - 能联系开票方补开或提供发票记账联复印件 + 开票方盖章证明的，凭复印件 + 证明报销。
  - 完全无法补开的（如对方已注销），需经办人写《发票遗失情况说明》，部门负责人签字，财务复核，**单笔 ≤ 500 元**可凭说明 + 其他佐证（付款记录、对账单）特殊处理；> 500 元的遗失原则上不予报销，除非财务经理特批。
- **不合规发票（常见情形）：**
  - 抬头错误（写了简称、写错税号）：退回让开票方红冲重开。
  - 税号缺失、品名笼统（如只写"办公用品一批"无清单）：要求补开清单或备注。
  - 发票超过有效期/跨年度未及时报（见场景 10）。
  - 票通查不到、查到状态异常（作废、红冲）：一律不予报销，疑似假票的移交财务并保留追责。
  - 连号、同一商家短期内多张拆分的小票：预警核查。

**单据：** 遗失说明、补开发票或证明、付款佐证。

**三类差异：**
- 外包的发票管理通常由供应商负责，外包个人遗失发票影响的是供应商对集团的结算，应第一时间找供应商财务补开，不要自己写说明绕过供应商。

**案例：** 内勤员工出差打车纸质发票洗衣服时洗烂了，金额 86 元。因为是滴滴，直接在滴滴 APP 重新下载了电子行程单和电子发票，重新上传搞定，根本不用走遗失说明。另一例：一张餐饮纸质发票 320 元丢了，餐馆能提供记账联复印件并盖章，凭复印件 + 盖章 + 当时的付款记录截图，写了遗失说明，部门负责人签字后报销成功。

## 场景 8：大额招待与事前报备

**规则：**
- 招待费 **> 5,000 元**必须事前在鼎财提"招待事前报备单"，写明：被招待单位、预计人数、预计金额、招待目的、是否涉及重要客户/政府关系（涉政府关系的另需合规法务确认）。
- 没有事前报备的大额招待，事后报销时财务可降档处理或拒付超额部分。
- 招待费有部门年度预算总盘，超预算的招待需分管领导特批。

**阈值：**
- ≤ 2,000 元：直属上级。
- 2,000 - 5,000 元：部门负责人。
- > 5,000 元：事前报备 + 分管领导。
- > 20,000 元：分管领导 + 财务总监 + 合规知会。

**单据：** 招待事前报备单（大额）、餐饮发票、被招待对象清单、事由说明。

**三类差异：**
- 研发对外招待较少，偶有招待来访合作院校/检测机构专家，同内勤标准。
- 外包**一律不可报招待费**（见 4.2 黑名单），不要尝试，必退。

## 场景 9：汇率与外币报销

**规则：**
- 外币费用统一折算为人民币入账。折算汇率口径：
  - 刷因公卡/公司信用卡的外币消费——以**银行实际入账人民币金额**为准（最优，无需自己折算）。
  - 个人现金/个人卡支付的外币——按**回国报销当日中国银行现钞买入价**折算，或采用财务每月公布的"月度记账汇率"，**以财务公告口径为准，全集团统一，不允许个人自选汇率**。
- 报销时外币金额、所用汇率、折算后人民币金额三者都要在鼎财明细里填清楚，并附汇率来源截图（中行牌价页面截图或财务公告）。

**阈值：** 汇率差异本身不设阈值，但外币报销往往伴随海外出差大额，按金额走对应审批。

**单据：** 外币原始票据 + 汇率截图 + 折算计算（建议附一张小的折算表）。

**踩坑提醒：** 不要拿信用卡账单上的金额和自己又按某个汇率折算的金额重复报，会出现金额对不上。刷卡的就认账单入账金额，现金的就按统一汇率，二选一，别混。

**案例：** 研发员工赴德国参加展会，住宿用因公卡刷了 600 欧元，账单实际入账 4,752 元人民币，报销就填 4,752 元，附银行账单截图。另有现金买的展会资料 50 欧元，按回国当日中行现钞买入价（假设 7.85）折算 392.5 元，附中行牌价截图。两笔合并报销，汇率口径清清楚楚，一次通过。

## 场景 10：跨年度费用与关账

**规则：**
- 集团每年 **12 月底关账**，原则上**当年费用当年报**。
- 每年 12 月通常会发"年终关账通知"，明确最后受理日期（一般是 **12 月 20 日前**提交当年费用报销，具体以当年通知为准）。
- 跨年度费用（去年发生、今年才报）的处理：
  - 金额较小（单笔 ≤ 1,000 元）且确有正当原因（如发票年底才到）的，可在次年 1 月并入处理，需说明。
  - 金额较大或无正当理由的跨年报销，需部门负责人 + 财务经理特批，财务可能要求做以前年度损益调整，比较麻烦，所以**强烈建议年内报完**。
  - **跨两个年度以上**（前年的费用拖到今年报）的，原则上不予受理，除非有特殊原因经财务总监批准。
- 关账期间（12 月下旬至次年 1 月初）系统可能临时关闭部分报销入口或限定提交，请关注关账通知。

**单据：** 正常单据 + 跨年说明。

**三类差异：**
- 研发费用跨年尤其要注意：研发费用归集和加计扣除是按纳税年度的，跨年研发费用错配会影响税务，费用会计会格外严格。所以研发员工的年内费用**务必赶在关账前报完**。
- 外包费用跨年的，应在供应商当年结算中处理，避免拖到次年。

**案例：** 内勤员工 12 月 28 日才拿到一张 11 月差旅的酒店发票（酒店开票慢），错过了 12 月 20 日关账提交。因金额 680 元且原因正当（发票延迟），写了说明，次年 1 月初并入处理，部门负责人和财务经理签字通过。如果这是一笔 2 万元的费用，那就要走以前年度损益调整，财务会很不情愿，所以平时拿到发票就赶紧报，别囤着。

## 场景 11：福利 / 节日 / 团建费用

**规则：**
- **节日福利**（端午、中秋、春节等）：由集团/人力统一发放（实物或福利卡），**不走个人报销**。部门不得自行采购节日礼品再报销，统一归口人力行政。
- **团建费用：** 部门团建有人均标准——**≤ 300 元/人/次**，每年原则上 **不超过 2 次**团建可报。团建需事前提"团建活动申请单"，写明人数、预算、活动内容，部门负责人审批；超人均标准或超次数的自付或特批。
- **生日会 / 小型部门关怀：** 挂"员工关怀费"，单月单部门 ≤ 500 元。
- **下午茶 / 节日茶歇：** 小额，挂行政杂费，注意不要频繁高额。

**阈值：** 团建总额按人数×300 封顶，> 10,000 元的大型团建到分管领导。

**单据：** 团建活动申请单、参与人数名单、餐饮/活动发票。报销时人数要真实，按实际参加人数核算人均，虚报人数套额度的查实严处。

**三类差异：**
- **外包员工不享受集团福利费和节日福利**（这是供应商与外包之间的关系）。团建是否邀请外包参加、费用谁承担，由项目和供应商约定；若约定由集团承担外包团建费，需外包管理办黄涛主任确认，且不计入集团福利、仅作项目费用处理。
- 研发同内勤。

**踩坑提醒：** 团建最常见的问题是人均超标和虚报人数。比如报了 30 人团建预算 9,000 元（人均 300），实际只去了 22 人，那人均就变成 409 元超标了，应按实际 22 人重算，超出部分要么自付要么砍掉。提交名单请填实际参加的人。

## 场景 12：客户垫付与返还

**规则：**
- **我方为客户垫付：** 因业务需要我方先替客户垫付了某些费用（如代客户支付的检测费、第三方服务费），这不是公司费用，是**应收款（往来款）**，不能当作费用报销冲掉，要挂"其他应收款—客户名称"，后续向客户收回。
- 垫付需事前经部门负责人 + 财务确认（确认确实是垫付且能收回），不能随便垫。
- **客户返还/客户报销给我方：** 客户把垫付款还回来时，对冲掉应收款，不是收入也不是费用减少，财务做往来核销。
- **客户给个人的返还绝对不允许进个人腰包**，必须全额上交公司对公账户。

**阈值：** 垫付 > 10,000 元需财务经理确认，> 50,000 元财务总监确认，并评估客户信用与回款风险。

**单据：** 垫付审批记录、垫付凭证、客户确认垫付的函件/邮件、回款记录。

**三类差异：**
- 外包/研发员工一般无权对客户做垫付，确有需要由集团对公处理。
- 涉及客户的资金往来非常敏感，任何"客户私下给现金/转账"的情况必须立即报告财务和合规，严禁个人截留。

**案例：** 某事业部应客户要求，先垫付了一笔 12,000 元的第三方认证费（认证最终给客户的产品用）。财务确认这是垫付、客户书面承诺承担，于是挂"其他应收款—某客户"。三个月后客户连同货款一起返还，财务核销往来款。整个过程没有走任何"费用报销"，因为这压根不是公司的费用。

## 场景 13：离职前费用清算

**规则：**
- **正式员工（内勤/研发）离职：**
  - 离职前必须把所有在途借款核销完毕、所有应报未报费用报完，否则离职手续（离职审批单/财务清算章）不予盖章。
  - 未核销借款从离职结算工资中代扣。
  - 离职后**原则上不再受理该员工的费用报销**，所以最后一笔费用务必在最后工作日前报完。确有遗漏（如离职后才到的发票），需原部门负责人 + 财务经理特批，且单笔 ≤ 1,000 元，超过的一般不再受理。
  - 公司发的资产（电脑、门禁卡、因公卡等）要在离职清单上逐项交还，因公卡未还清的挂账从结算中扣。
- **外包员工离场：**
  - 离场前由甲方项目经理 + 供应商对接确认所有可报费用，**离场当月内**通过供应商结算处理完毕。
  - 外包离场后**不再受理对私报销**（见 4.5），所有遗留费用走供应商结算闭口。
  - 外包归还的项目资产、门禁、设备需在离场清单上确认。

**阈值：** 离职清算无金额阈值，但代扣借款、资产挂账金额从结算工资中扣除，超过结算工资的部分需个人补缴。

**单据：** 离职/离场清单、借款核销记录、资产交还确认、最后费用报销单。

**三类差异：** 见上，正式员工和外包的清算口径不同，核心区别是外包通过供应商闭口、离场后对私彻底关闭。

**案例：** 研发员工小王离职，离职清算时发现他还有一笔 5,000 元出差备用金没核销。他赶在最后工作日把实际差旅 4,300 元报了，系统冲销借款，应退公司 700 元，从离职结算工资里扣。同时他交还了笔记本电脑和因公卡（因公卡有 200 元未还消费，一并扣除）。财务清算章盖了，离职手续才走完。如果他离职后才想起来还有一笔 800 元没报，原部门负责人 + 财务经理特批后还能处理；但若是 3,000 元，对不起，离职后基本不再受理，损失自负。

## 附录 C：常见退单理由 TOP 10（自查清单）

1. 发票抬头写了简称"鼎垣"，没写全称"鼎垣智造集团有限公司"。
2. 票通查不到 / 发票已作废红冲 / 发票重复报销。
3. 招待只写"招待客户"，没写单位、人数、事由。
4. 研发专项没填项目编号。
5. 研发设备 ≥5,000 元当费用报，没走资产采购。
6. 外包报了招待/福利/通讯，或超过外包白名单范围。
7. 外包没有甲方对接人确认就提交。
8. 大额本该对公却拆成多张个人报销单（化整为零）。
9. 出差没有事前出差申请单编号。
10. 借款超 30 天未核销 / 跨年度费用无说明。

## 场景 14：政府补助 / 专项资金项目的费用

这两年集团拿了几笔政府补助——技改专项、首台套、研发补助、人才专项之类的。这类钱跟自有资金完全不是一回事，花起来比平时严十倍，因为后面要接受财政、审计、第三方专项审计的检查，花错一笔可能整个项目验收都过不了，甚至要退钱。

**第一条铁律：专项资金必须专款专用、单独核算。** 凡是用政府补助、专项资金支付的费用，在鼎财云建单时必须挂对应的**专项项目号**（比如"ZX-2026-技改-007"这种格式，立项时由财务专项会计统一编），不能跟日常运营费用混在一个成本中心里。金蝶那边这些专项是单独设科目、单独建辅助核算的，年底要单独出"专项资金使用情况表"报给主管部门。所以你报一笔专项费用，**项目号填错或者没填，系统直接卡住提交**，跟研发专项一个道理。

**第二条：只能花"预算书里写了的、批文允许的"。** 政府补助申报时都附了一份"资金使用预算"，钱怎么花、花在哪几个科目、各占多少，批文里是认过的。你不能拿技改补助去报招待费、去发福利、去买跟项目无关的办公家具。常见的硬性负面清单：

| 不允许用专项资金列支的 | 原因 |
|---|---|
| 业务招待费、礼品、酒水 | 财政资金一律不得用于招待，碰都别碰 |
| 罚款、滞纳金、违约金 | 不得列支，自有资金都不能税前扣，专项更不行 |
| 与项目无关的差旅、办公、福利 | 专款专用，串用即挪用 |
| 应由自有资金承担的日常运营开支 | 不能拿补助补运营窟窿 |
| 超预算科目的支出 | 预算书没列的科目原则上不能开支，确需调整要报主管部门批 |

**第三条：单据比平时还要全。** 专项资金的发票，除了正常的票通验真、抬头全称这些，还要求：合同（哪怕几千块的服务也要有合同或协议）、付款凭证（最好对公转账，能不走现金就不走现金）、验收/成果证明、以及这笔费用跟项目的关联说明。专项审计的人最爱问的一句话是"这笔钱跟你这个项目什么关系"，你得拿得出书面的东西回答。

**审批：** 专项费用在常规双线之外，**财务线必须加签财务专项会计**这一道（确认在预算口径内、科目对、不超概算），大额（单笔 ≥ 50,000 元）的还要财务总监周敏知会。业务线这边专项项目负责人是必签的，跟研发的项目负责人一个意思。

**真实案例（驳回）：** 某事业部拿了一笔设备技改补助，项目组想用补助里的钱请来验收的专家组吃个饭，报了 2,800 元餐费挂在专项项目号下。财务专项会计直接退，理由：财政资金不得列支招待费，这是红线，跟金额大小没关系。后来这顿饭改用部门自有的招待费预算列支，跟专项资金彻底切开，这才合规。教训：**专项资金里一分钱招待都不能有**，要请客用自己的钱。

**再一个（部分准予）：** 研发补助项目里报了一台 4.8 万的测试设备，发票、合同、验收都齐，但预算书里这一项原报的是 4 万。超的 8,000 元怎么办？专项资金只认 4 万（预算内），超出的 8,000 元用研发自有预算补上，分两笔记账。所以专项采购**一定要在预算概算之内**，超了就得自掏。

## 场景 15：研发费用加计扣除的归集

这一节是写给研发员工、研发中心管理岗，以及帮研发记账的同事看的。手册第 3 节讲了研发专项怎么报，场景 10 也提了一句研发跨年要小心，但"为什么卡这么死""归集到底归的是什么"一直没讲透。这次讲明白，因为加计扣除是实打实给集团省税的，归集做对了是"花对了钱还省了税"，做错了轻则加计不上，重则被税务认定多扣要补税加滞纳金。

**先说清楚加计扣除是什么。** 简单讲，国家为了鼓励企业搞研发，允许你把符合条件的研发费用，在已经据实扣除的基础上**再额外多扣一块**（具体加计比例以当年税收政策为准，制造业近年是相当高的比例），等于变相少交企业所得税。但前提是：这笔费用得是"真研发费用"，得能归集到具体研发项目，得有完整的辅助账。换句话说，**钱花了不等于能加计，得归集对了才算。**

**哪些费用能进研发费用归集（高频的）：**

| 能归集的研发费用 | 报销时怎么挂 |
|---|---|
| 直接从事研发人员的人工费用 | 走工资系统，但研发工时要跟项目对得上（见下文） |
| 研发用的材料、试剂、耗材、样品 | 必须挂 RD 项目号，附领料清单/采购清单 |
| 研发设备折旧（专用于研发的） | 资产入账时标"研发专用"，由资产会计归集 |
| 委托外部机构做的研发（外协、检测、试制） | 必须有委外合同，按规定比例归集 |
| 与研发活动直接相关的差旅、会议（有限额、占比卡得严） | 挂 RD 项目号，写清是去干研发相关的什么事 |

**哪些花了也加计不上（最容易混的）：**

- 研发人员去市场调研、跑客户、做销售支持的差旅——这不是研发活动，挂 RD 也归集不进去。
- 借研发名义买的通用设备、通用办公电脑——通用电脑不是研发专用设备，加计不了，还可能被认定虚增研发费用。
- 跟生产经营混用、分不清是研发还是生产的费用——分不清的一律不能全额进研发归集。
- 行政管理人员的工资、一般管理费用——不属于研发人工。

**两条操作上的硬要求：**

1. **工时要对得上。** 研发人员如果同时干研发又干别的（比如又支持生产、又跑项目），他的人工费用要按**研发工时占比**拆分，只有研发那部分能归集。所以研发中心现在要求填工时表，工时表数字要跟研发费用归集对得上。你报销时挂的项目、填的工时，财务和内审是会拿来交叉核对的，对不上就有问题。

2. **日常运营预算和研发专项预算绝对隔离，不许互相调。** 这一条制度正文里也写了，这里再强调一遍：因为加计扣除盯的就是研发专项这个盘子，你要是把运营费用调进研发预算、或者把研发费用拿去补运营，归集口径就乱了，加计扣除的基数也就不可信了。**研发的钱和运营的钱，中间那堵墙不能拆。**

**真实案例（准予，正面典型）：** 研发员工买了一批测试用元器件 1.2 万元，取的是增值税专票，挂对了 RD-2026-018 项目号，附了领料记录。这笔：进项税正常抵扣，费用归集进研发辅助账参与加计扣除。花对了钱、抵了进项、还省了所得税，三件好事。

**真实案例（驳回）：** 某研发组报销时把一台 6,000 元的通用笔记本电脑报成"研发测试设备"，想挂进研发归集。费用会计退单：通用电脑不属于研发专用设备，加计不了；而且 ≥ 5,000 元要走资产流程不能当费用报。这种"借研发名义采购通用物资"是内审专门盯的点，被抓到要记异常。

**真实案例（驳回，归集错了）：** 一笔研发外协测试费本该挂 RD-2026-0123（这个项目有预算且在归集范围），结果挂成了 RD-2026-0098（这个项目预算已经用完）。退回重挂。原因不光是预算，更要命的是：挂错项目会让两个项目的研发费用归集都失真，年底加计扣除按项目归集时全乱套。**研发项目号填错，比金额报错严重得多。**

## 场景 16：外包结算、发票与个人所得税

手册第 4 节把外包怎么报、白名单、对私红线讲得很细了。这一节补的是结算和税的层面——也就是钱最终怎么从集团流到供应商、再到外包个人，中间哪些地方有个税的雷。这块最容易踩坑，别问我为什么知道。

**先把三种"外部人力"的钱分清楚，对应不同的税：**

| 类型 | 跟谁签合同 | 谁开发票 | 个税谁管 |
|---|---|---|---|
| 外包/驻场（W 工号） | 集团 ↔ 供应商 | 供应商对集团开服务费发票 | 供应商代发工资、供应商管个税，集团不直接代扣 |
| 个人零星劳务（临时找人讲课、设计、翻译） | 集团 ↔ 个人 | 个人去税务代开发票 | **集团付款时必须代扣代缴个人劳务报酬个税** |
| 个人长期但实质雇佣 | —— | 这种应该走工资，别拿劳务规避社保个税 | —— |

**最大的雷：对私付给个人劳务，不代扣个税。** 财务这边一看到"付给某个自然人""劳务费""咨询费"，第一件事就是查有没有代扣个税。劳务报酬是法定要代扣代缴的，你直接对私把全额打给人家、没扣税，这就是税务大雷，被查到集团要补税、交滞纳金、还可能罚款。所以：

- 找外部个人干活（讲课、设计、做方案这种零星劳务），对方去代开发票，集团付款时**按规定预扣个人劳务报酬个税**，留好代扣记录。
- 报销/付款单里涉及对私付给自然人的，必须能看到个税代扣这一栏，**没有代扣记录的，费用会计一律退**。

**外包结算和"两头报"的防范。** 外包默认走供应商月度结算（手册 4.4 方式 A），外包个人发生的可报费用汇总进供应商当月对账单，供应商对集团开票。这里有个高发问题：**同一笔费用，外包个人在鼎财报了一次，供应商又把它算进服务费报了一次，两头收钱。** 比如驻场外包某天加班餐 48 元，他在客户那边领了餐补，又跑回鼎财报了一次。

防这个的办法是供应商对接人那道签字——他签字的含义就是"这笔费用属于合同约定范围、且不会在供应商服务费里重复"。所以这道签字不是走过场。月度结算时，财务还会拿外包个人报销的明细跟供应商的服务费结算单做交叉核对，对不上的拎出来查。

**真实案例（驳回 + 严重，涉供应商）：** 某驻场外包一笔现场加班餐 48 元，既在客户处领了餐补，又在鼎财报了一次。金额很小，但性质是两头报、套现、造假。处理：追回，记入该外包不良记录，通报供应商；甲方对接人因为没尽到"无重复报销"的确认义务，连带提醒。**金额小不等于性质轻**——48 块钱的两头报，跟 4,800 块的造假，定性是一样的。

**真实案例（驳回）：** 找了个外部专家来做一天技术评审，付劳务费 8,000 元，对方代开了发票，但经办人填单时图省事直接走对私、没填个税代扣。退单，必须补上劳务报酬个税的代扣计算和扣缴记录才能付。教训：**对个人付劳务，先想个税，再想付款。**

## 场景 17：客户返点 / 返利的处理

集团这两年跟一些大客户、渠道有返点返利的约定——年度采购量到了某个台阶，客户返我们一笔；或者反过来，我们给下游一笔返利。这个跟报销不直接相关，但经办的业务同事经常搞不清，把返点当成"收到一笔钱"或"花出去一笔钱"随便处理，账就错了，税也错了。这里讲清楚原则。

**第一条：返点返利绝对不能进个人腰包，必须全额走对公。** 跟手册场景 12 客户垫付返还一个道理——**任何客户或供应商私下给个人现金、给个人转账的"返点""好处费""感谢费"，一律视为违规，必须立即报告财务和合规，全额上交对公账户。** 这条是底线中的底线，谁碰谁出事。

**第二条：我方收到的返点，一般是冲减成本或确认收入，不是"天上掉的钱"。**

- 如果是采购返利（我们买东西，供应商按量返我们钱）：通常**冲减采购成本**，供应商要开红字发票或者按规定处理，不能简单当成一笔营业外收入入账，更不能不入账。
- 如果是销售返利（我们作为渠道，上游因我们的销售业绩返给我们）：按约定确认相应收入或冲减，涉及开票，财务统一处理。

**第三条：我方给客户的返利，要有合同依据，且涉及发票和税。**

- 给下游客户的返利、折扣，必须在销售合同或返利协议里写清楚条件、比例、结算方式。
- 在同一张发票上的折扣，能税前认；事后另付的返利，要按规定开票或取得对方发票，没有合规票据的返利不能税前扣除。
- **绝对不能用"返点""市场支持费"的名义，行商业贿赂之实**——给客户经办人个人好处、虚构市场费用套现，这是合规和刑事红线。

**第四条：报销系统里基本不处理返点。** 返点返利是收入/成本的事，走的是销售、采购和财务的合同结算流程，不是费用报销。如果你在鼎财里建了个单子想"报销"一笔返点，那基本就是搞错了，找财务的销售会计或往来会计处理。

**真实案例（拦截 + 报合规）：** 某业务同事跟客户对接，客户私下说"这次给你个人 5,000 元辛苦费"。该同事第一时间报告了部门负责人和合规。处理：明确告知这笔钱若有，必须走对公、计入相应往来或收入，个人绝不能收。最终客户的返点按合同走了对公结算。教训：**客户给个人的任何钱，先报告，别先收。**

**真实案例（驳回 + 提醒）：** 某事业部想给一个渠道客户一笔 3 万元返利，业务同事在鼎财建了张"市场推广费"报销单，没有返利协议、没有对方发票，想直接对私打给客户的联系人。退单，理由：返利必须有协议依据、走对公、取得合规票据；对私打给客户个人涉嫌商业贿赂。后来补了返利协议、走对公、客户公司开了发票，才合规处理。

## 场景 18：跨主体（子公司之间）费用借用与结转

集团下面好几个子公司、事业部独立核算，承接客户项目的时候经常出现 A 子公司的员工帮 B 子公司的项目花了钱，或者 A 替 B 垫付了一笔。这类"跨主体"的事最容易把账搞乱，因为发票抬头、进项抵扣、成本归属、子公司之间的内部往来全牵扯进来。这一节给个清楚口径。

**第一条，也是最关键的一条：发票抬头要开对主体。** 谁受益、费用该归谁，发票就开谁的全称抬头。B 子公司项目用的费用，发票就该开 B 子公司的全称（每个主体都有自己的工商全称和税号，鼎财首页公告里有各主体抬头对照表，建单时选对主体，系统会带出对应抬头）。

**开错主体的后果（讲明白大家就长记性了）：**
- 发票开成了 A 的抬头，但费用实际是 B 项目的——A 这边凭空多了一张跟自己业务对不上的票（进了 A 的账，A 的成本虚增，进项还可能被质疑），B 那边该有的成本和进项却没有（B 抵不了这张票的进项税）。两头都是问题。
- 这种票**在 B 主体下报不了**（抬头不是 B），在 A 主体下报又跟 A 的业务对不上。所以经办人会卡在那里两头不是人。

**第二条：A 员工确实先垫付了 B 主体的费用怎么办？** 走**内部往来**，不当 A 的费用。具体：

- 发票抬头开 B 主体（前提还是上面那条，抬头先开对）。
- A 这边垫付的钱，挂"其他应收款—B 子公司"（A 的往来挂账），不进 A 的费用。
- 费用实际计入 B 主体（B 确认这笔费用、入 B 的成本、抵 B 的进项）。
- A、B 之间通过**内部往来结算**把这笔钱划平，按集团内部结算规则定期清理，不能长期挂着不平。

**第三条：跨主体费用结转必须有依据、要双方确认，且不能用来调利润。** 把一笔费用从一个主体转到另一个主体，必须填"费用调整单"（或内部往来结算单），写清原主体、目标主体、原因、金额、对应的项目，由两边的财务复核确认。**严禁拿跨主体结转来调子公司利润**——比如把亏损子公司的费用转到盈利子公司去消化，让两边报表都好看，这是操纵利润，内审专门查。

**第四条：跨主体的内部往来要定期清理，不许变成"长期挂账"。** 内部往来挂账超过约定账期（一般跟着月度/季度结算走）没清的，列入往来清理清单；金额大的、挂太久的，财务会专门拎出来对账，该结的结、该收的收。长期挂账既影响各主体报表真实，也是审计的关注点（详见场景 20 长期挂账）。

**审批：** 跨主体的费用借用、结转，业务线要两边的项目/部门负责人都认；财务线两边财务复核都要签；**结项后或大额（≥ 50,000 元）的跨主体结转，要财务经理审批。**

**真实案例（驳回，开错主体）：** 鼎垣某子公司 A 的工程师去支援子公司 B 承接的客户项目，在现场买了批耗材 3,200 元，发票顺手开成了 A 的全称抬头，回来想在 A 名下报。退单：这笔是 B 项目的费用，抬头开错了。处理：发票找供应商红冲重开成 B 的抬头，费用计入 B，A 这边垫付的钱挂"其他应收款—B 公司"，月度内部往来结算时划平。教训：**跨主体花钱，先确认这是哪个主体的事，再让供应商开对应抬头的发票，别图顺手。**

## 场景 19：混合用途费用的拆分

一笔费用里既有能这么算的、又有能那么算的，混在一起，这种"混合用途"的单子是审单的高发退单区。最典型的是一张发票里既有能抵进项又有不能抵的，还有一笔费用横跨几个项目/部门的。手册第二部分零散提过分摊，这里集中讲拆分的几种情形和规则。

**情形一：一张发票里既有可抵进项又有不可抵的。** 最常见的是出差住宿——一张携程或酒店开的发票里，住宿费（增值税专票部分可按规定抵扣）和餐饮费（餐饮进项不得抵扣）混在一起。规则：**在鼎财明细里按品名拆成多行，分别选对科目和税率**，别整张当住宿报。住宿走住宿、按可抵处理；餐饮单列一行、按不可抵处理。平台的"代订服务费"再单列一行（这部分一般 6% 可抵），别拿服务费冒充住宿费。

**情形二：一笔费用同时服务好几个项目/部门（共用费用）。** 比如一趟差跑了三个客户项目、一台共用测试设备、驻场场地的水电。规则：**按合理的分摊基准拆到各项目/成本中心**，每行写清分摊基准和依据：

| 共用费用类型 | 分摊基准 |
|---|---|
| 一趟差跑多个项目的城际交通（机票/高铁） | 按各项目实际工作日数比例 |
| 多项目共用的住宿 | 按为各项目过夜的天数 |
| 共用人员的人力/差旅 | 按项目工时占比 |
| 共用设备/测试台 | 按使用机时（台时）占比 |
| 共用场地/水电 | 按面积或人头 |

**铁律：同一类共用费用，一个项目周期内只能用一种分摊口径，定了就不许中途换；用了哪种要在备注里写明。** 中途换口径（比如这个月按工时、下个月按天数）会触发系统预警，因为这往往是有人想调项目毛利。分摊比例还会跟考勤/工时系统勾稽，偏差太大（制度定的是超 20% 自动预警）会被拎出来核。

**情形三：因公因私混在一起。** 出差顺便办了私事、住宿多住了一晚是私人行程、机票里夹了配偶的票——这种**因私的部分必须自己剔出来自付**，只报因公那部分。整张混报被发现，轻则退回剔除，重则按虚报处理。

**真实案例（部分准予）：** 一笔机票 2,400 元，是一趟差跑了三个客户项目，经办人图省事平均分三份各 800 元挂三个项目。退回重报：要按到访天数分（实际是 2:1:1），重新拆成 1,200 / 600 / 600 挂三个项目，并在备注写明依据。平均分不是"公平"，是没依据。

**真实案例（驳回重报）：** 一张住宿专票里含住宿 6%（可抵）和餐饮（不可抵），经办人整张当住宿报、把餐饮也按可抵处理了。退单：必须按品名拆成多行、分别选科目，餐饮那部分进项要剔出来不抵。

## 场景 20：长期挂账与预付款的清理

挂账这事，挂的时候都觉得"先记着，回头处理"，结果一挂就是大半年没人管，月底季度末财务对账时一抓一大把。长期挂账是审计最爱盯的，因为它后面往往藏着该核销没核销、该收回没收回、甚至虚增资产的问题。这一节讲清楚几类挂账的清理时限和规则。

**先认清几类常见的"挂账"：**

| 挂账类型 | 什么意思 | 该怎么清 |
|---|---|---|
| 个人借款/备用金 | 借了钱出差/办事还没核销 | 活动结束后规定时限内核销，多退少补 |
| 对公预付款 | 预付了供应商定金/货款，东西/服务还没到 | 到货/完工凭发票验收核销，挂"预付账款" |
| 暂估入账 | 东西收到用了、发票还没来，先按合同估 | 发票来了冲销暂估 |
| 其他应收款（垫付） | 替客户/兄弟主体垫的钱 | 收回时核销往来 |

**清理时限（写死，这次统一口径）：**

- **个人借款/备用金：** 一笔未核销不得借第二笔（系统卡）；超 60 天冻结新借款；离职前必须全部核销。这些跟手册场景 6 一致，不重复。
- **对公预付款：** 挂"预付账款"，**不进费用**。超 **90 天**未核销的逐笔清查、列账龄异常清单；超 **180 天**未核销的，冻结该供应商的新对公付款，必须把老的预付清了才能再付新的。
- **暂估入账：** 暂估挂账**超 3 个月未冲销**的，列月度内控检查。暂估是临时的，发票一来就得冲，不能拿暂估长期挂着不办。
- **其他应收款/垫付：** 单客户在途垫付累计超 **10 万**再新增垫付要事业部总监批；超 **90 天**列月度催收；超 **180 天**评估坏账、上法务。

**为什么卡这么死？** 三个原因：一是长期挂账往往意味着钱收不回来或者事没办成，越拖越烂；二是预付款长期不冲，资产是虚的，报表不实；三是有人专门用挂账来"藏"问题——比如该报的费用迟迟不报、用暂估顶着，或者垫付了根本收不回的钱长期挂应收款不计坏账。所以财务月底跑账龄，超期的一律要么核销、要么书面说明，没有第三种选择。

**操作上：** 鼎财云每月末会跑一张"超期挂账清单"——超 60 天未核销借款、超 90 天未核销预付、超 3 个月未冲暂估、超 90 天未收回垫付，全列出来推给对应经办人和部门负责人，限期处理；处理不了的要写说明。**僵尸预申请**（事前审批通过后预占了预算额度、但 60 天都没有对应报销上来）系统会自动释放预占，把额度还回去，免得占着茅坑。

**真实案例（驳回 + 代扣）：** 某员工借了 2 万备用金出差，回来只报了 8,000，剩下 1.2 万既不退也不报，挂在"其他应收款"上超了 90 天。处理：从工资代扣，并计入个人信用记录，暂停其新借款。教训：借了多少、花了多少、剩多少退回，是分分钟的事，别拖成挂账。

**真实案例（账龄异常清查）：** 某事业部给一家供应商预付了 8 万定金，结果项目变更供应商没供货，钱也没退回来，预付账款挂了 200 多天没人管。财务跑账龄抓出来：超 180 天，冻结该供应商所有新对公付款，责令业务部门要么追回预付款、要么走合同违约处理，限期清账。

## 场景 21：离职 / 调岗交接时的费用清算

手册场景 13 讲了离职前费用清算的基本规则——借款核销完、应报费用报完、资产交还、否则不盖财务清算章。这一节补两块手册没细讲的：一是**调岗（不是离职，是内部换岗/换部门/换项目）**时费用怎么交接，二是离职/调岗时**在途单据和挂账**的处理，因为人一走，单子卡在审批流里、挂账没人认领，是非常实际的麻烦。

**先说调岗。** 内部调岗的人不离职，但他原来负责的项目、成本中心、审批权限都要交接。常见的坑：

1. **在途报销单（他作为经办人提交、还没走完审批的）：** 这些单子继续走完即可，审批层级以**提交时点**他的岗位锁定（制度正文里写的，审批中岗位变动不影响在途单），所以调岗不影响他自己已经提交的单子最终能批多少。
2. **他作为审批人手里待办的单子：** 调岗当日，HR 在系统里把他的待办**自动转给继任者或上级**，别让单子卡在一个已经不在这个岗位的人手里。这块鼎财跟 HR 系统是同步的，调岗生效当天自动转。
3. **他经手但还没核销的借款、挂账、预付：** 调岗**不等于**这些可以甩给后任不管。原则上谁借的款谁核销、谁经手的挂账谁交接清楚。调岗前要做一张"经办事项交接清单"，把在途借款、未核销预付、未清挂账、未报费用列清楚，跟继任者交接、双方签字，财务备案。

**再说离职，补充几个手册没细说的点：**

- **离职前必须做"费用清算 checklist"**，逐项打勾：所有在途借款核销了没？所有应报未报费用报完了没？经手的对公预付清了没？经手的客户垫付（其他应收款）交接清楚了没？因公卡/项目资产交还了没？这张表过不了，财务清算章不盖。
- **在途单据：** 离职生效日，他作为审批人的待办自动转继任者/上级；他作为经办人已提交的单子继续走完（最后费用务必赶在最后工作日前提交，离职后原则上不再受理，确有遗漏单笔 ≤ 1,000 元且原部门负责人 + 财务经理特批才处理）。
- **挂账与代扣：** 未核销借款、因公卡未还消费、资产未还折价，统统从离职结算工资里扣；结算工资不够扣的，从经济补偿、未结报销、应发奖金里抵；再不够，签还款承诺，逾期移法务。
- **涉及项目奖金的费用要算清。** 离职/调岗的人如果经手过参与项目奖金核算的费用，这些费用是否真实、是否归对项目，直接影响项目毛利和奖金。所以交接时这类费用要重点核——别让一个人调岗/离职，把一堆挂错项目的费用留给后任背锅。

**外包离场（补充场景 13）：** 外包离场前由甲方项目经理 + 供应商对接确认所有可报费用，离场当月内通过供应商结算闭口；离场后不再受理对私报销，遗留费用全走供应商结算。外包归还的项目资产、门禁、设备在离场清单上逐项确认。

**真实案例（调岗交接没做清，后任背锅）：** 某项目组长调去别的事业部，走的时候有一笔 1.8 万的供应商预付款没核销、还有两笔挂错项目的差旅没调正，他没做交接清单就走了。三个月后财务跑账龄抓出来，找继任者，继任者一脸懵——这些都不是他经手的。最后费了好大劲把原经办人请回来对账。教训：**调岗交接清单不是形式，在途借款、挂账、未报费用、错挂项目，一笔笔列清楚交接签字，不然就是给后任挖坑。**

**真实案例（离职清算，代扣 + 特批）：** 研发员工离职，清算时发现一笔 5,000 元备用金没核销。他赶在最后工作日报了实际差旅 4,300 元，系统冲销，应退公司 700 元从结算工资扣；同时交还笔记本电脑和因公卡（因公卡 200 元未还消费一并扣）。财务清算章盖了，手续走完。离职后他才想起一笔 800 元没报，原部门负责人 + 财务经理特批后处理了；但如果是 3,000 元，离职后基本不再受理，损失自负。

| 跨主体费用 | 发票开对受益主体、垫付挂内部往来 | 开错主体两头报不了、拿结转调利润 |
| 混合用途拆分 | 按品名/按基准拆成多行、一周期一种口径 | 整张当一种报、平均分摊无依据、因私混报 |
| 长期挂账与预付清理 | 预付超90冻、暂估超3月查、垫付超90催 | 预付长期挂不冲、暂估顶着不办、借款不核销 |
| 离职/调岗清算 | checklist逐项打勾、交接清单签字 | 调岗不做交接甩锅后任、离职后才补报大额 |

最后一句还是那句老话：底下这些事，不确定就问——制度口径问费控科郑国强，审单退单问报销审核科李文娟，专项资金和研发归集问财务专项会计这条线，跨主体和往来问财务往来会计，外包问黄涛主任，大额疑难问到周敏总那也行。问清楚再做，比做错了返工、比把税务和子公司账搅乱强一百倍。

——集团财务部（费控科、报销审核科） 2026 年补编
