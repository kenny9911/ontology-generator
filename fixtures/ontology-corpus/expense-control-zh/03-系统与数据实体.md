# 系统与数据实体说明

> 文档编号：DATA-EXP-003
> 说明：定义费控与报销领域的记录系统（System of Record）及其核心数据实体、属性、主外键、枚举值、系统集成与系统间交换事件。命名与其余三份文档严格一致。

---

## 第一部分 业务系统（记录系统）

| 系统代号 | 系统全称 | 职责 | 承载实体 |
|---|---|---|---|
| **ECS** | 费控报销平台 | 报销主流程、工作流引擎、合规引擎 | 报销单、报销明细、预申请单、审批节点、审批规则、合规预警、驳回记录 |
| **BMS** | 预算管理系统 | 预算编制、下达、预占、核销 | 预算、成本中心 |
| **IVC** | 发票查验中心 | OCR、发票真伪、抬头校验、重复校验 | 发票 |
| **GL-ERP** | 财务核算系统 | 会计凭证、入账、总账 | 会计凭证（派生）、成本中心映射 |
| **TPS** | 资金支付系统 | 付款执行、账期管理 | 付款单 |
| **IAM-HR** | 统一身份与组织系统 | 员工、部门、岗位、汇报关系主数据 | 员工、部门 |
| **DAS** | 影像归档系统 | 单据与影像不可篡改归档 | 归档记录（派生） |

记录系统主数据归属：员工与部门主数据以 IAM-HR 为准；成本中心与预算以 BMS 为准；供应商主数据以 GL-ERP 为准；发票以 IVC 为准；报销单据以 ECS 为准。

## 第三部分 系统集成（接口）

1. **IAM-HR → ECS**：每日同步员工（含 employee_type、manager_id）与部门主数据，用于审批路由。
2. **BMS ↔ ECS**：BMS 推送已下达预算余额至 ECS；ECS 回调 BMS 执行预算预占（ReserveBudget）与核销（SettleBudget）。
3. **ECS → IVC**：ECS 上传发票影像调用 IVC 的查验接口（VerifyInvoice），同步返回结构化字段与查验结论。
4. **ECS → GL-ERP**：财务复核通过后 ECS 推送凭证信息至 GL-ERP 生成会计凭证（PostToGL）。
5. **ECS → TPS**：ECS 推送付款单（CreatePaymentOrder）至 TPS；TPS 回调付款结果。
6. **ECS → DAS**：归档报销单、发票影像与审批记录（ArchiveClaim）。

## 第四部分 系统间交换的事件

| 事件 | 生产系统 | 消费系统 |
|---|---|---|
| 预算已下达事件（BudgetIssued） | BMS | ECS |
| 发票已查验事件（InvoiceVerified） | IVC | ECS |
| 报销单已审批事件（ClaimApproved） | ECS | GL-ERP、TPS |
| 会计凭证已生成事件（VoucherPosted） | GL-ERP | ECS |
| 付款已完成事件（PaymentCompleted） | TPS | ECS、BMS |
| 单据已归档事件（ClaimArchived） | DAS | ECS |
