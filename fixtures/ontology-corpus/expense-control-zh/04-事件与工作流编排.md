# 事件目录与工作流编排

> 文档编号：FLOW-EXP-004
> 说明：定义费控与报销领域的事件目录（含载荷、生产动作、消费动作）与端到端工作流定义。所有实体、系统、动作、事件命名与其余三份文档严格一致。

---

## 第二部分 事件目录（Events）

### E-01 预算已下达事件（BudgetIssued）
- 载荷：budget_id、cost_center_id、category_id、fiscal_year、total_amount、available_balance
- 产生动作：IssueBudget
- 消费动作：SyncBudgetBalance

### E-02 预申请已提交事件（PreApprovalSubmitted）
- 载荷：preapproval_id、applicant_id、employee_type、estimated_amount、category_id
- 产生动作：SubmitPreApproval
- 消费动作：RouteApproval

### E-03 预申请已批准事件（PreApprovalApproved）
- 载荷：preapproval_id、reserved_budget_id、estimated_amount
- 产生动作：ApprovePreApproval
- 消费动作：ReserveBudget

### E-04 预申请已驳回事件（PreApprovalRejected）
- 载荷：preapproval_id、node_id、reason
- 产生动作：ApprovePreApproval
- 消费动作：SubmitPreApproval（重新提交）

### E-05 发票已查验事件（InvoiceVerified）
- 载荷：invoice_id、invoice_code、invoice_no、amount、verify_result
- 产生动作：VerifyInvoice
- 消费动作：AddClaimLineItem

### E-06 报销单已提交事件（ClaimSubmitted）
- 载荷：claim_id、applicant_id、employee_type、total_amount、occurred_date
- 产生动作：SubmitExpenseClaim
- 消费动作：RunComplianceCheck、RouteApproval

### E-07 合规预警事件（ComplianceAlertRaised）
- 载荷：alert_id、claim_id、alert_type、severity
- 产生动作：RunComplianceCheck、VerifyInvoice
- 消费动作：ApproveExpenseClaim（审批人确认预警）

### E-08 超预算阻断事件（BudgetBlocked）
- 载荷：claim_id、budget_id、requested_amount、available_balance
- 产生动作：RunComplianceCheck
- 消费动作：SubmitExpenseClaim（退回申请人修改）

### E-09 审批节点已通过事件（NodeApproved）
- 载荷：node_id、claim_id、approver_id、sequence
- 产生动作：ApproveExpenseClaim
- 消费动作：ApproveExpenseClaim（流转下一节点）

### E-10 报销单已审批事件（ClaimApproved）
- 载荷：claim_id、total_amount、payment_type
- 产生动作：ApproveExpenseClaim
- 消费动作：FinanceReview

### E-11 报销单已驳回事件（ClaimRejected）
- 载荷：claim_id、node_id、reason
- 产生动作：ApproveExpenseClaim、FinanceReview
- 消费动作：SubmitExpenseClaim（补充重提）、RaiseAppeal

### E-12 审批超时升级事件（ApprovalEscalated）
- 载荷：node_id、claim_id、original_approver_id、escalated_to_id
- 产生动作：ApproveExpenseClaim（超时检测）
- 消费动作：ApproveExpenseClaim（上级处理）

### E-13 财务复核已通过事件（FinanceReviewed）
- 载荷：claim_id、reviewer_id、payment_type
- 产生动作：FinanceReview
- 消费动作：PostToGL、CreatePaymentOrder

### E-14 会计凭证已生成事件（VoucherPosted）
- 载荷：claim_id、voucher_no、cost_center_id、amount
- 产生动作：PostToGL
- 消费动作：ArchiveClaim（凭证纳入归档）

### E-15 付款已完成事件（PaymentCompleted）
- 载荷：payment_id、claim_id、payee_type、amount、pay_date
- 产生动作：ExecutePayment
- 消费动作：SettleBudget、ArchiveClaim

### E-16 单据已归档事件（ClaimArchived）
- 载荷：claim_id、archive_id、archived_at
- 产生动作：ArchiveClaim
- 消费动作：（流程终结）

### E-17 预占已释放事件（ReservationReleased）
- 载荷：preapproval_id、budget_id、released_amount
- 产生动作：SettleBudget（超时释放）
- 消费动作：SyncBudgetBalance

### E-18 申诉已发起事件（AppealRaised）
- 载荷：appeal_id、claim_id、applicant_id
- 产生动作：RaiseAppeal
- 消费动作：ResolveAppeal

### E-19 申诉已裁决事件（AppealResolved）
- 载荷：appeal_id、claim_id、decision
- 产生动作：ResolveAppeal
- 消费动作：（终结或回流审批）

## 第三部分 工作流定义（Workflows）

### WF-01 预算编制与下达流程
- 触发事件：年度预算编制启动（定时触发）
- 编排策略：顺序编排
- 参与角色：预算管理员、分管领导、BMS、ECS
- 步骤：IssueBudget →（产生 BudgetIssued）→ SyncBudgetBalance
- 结果：ECS 建立预算可用余额实时账，供后续报销校验使用。

### WF-02 大额费用预申请流程
- 触发事件：申请人预估费用达到事前审批阈值
- 编排策略：顺序 + 分支（按员工类型路由）
- 分支条件：
  - 内勤员工 → 直属上级 →（≥20,000 追加财务专员）部门负责人
  - 研发员工 → 直属上级 → 研发部门负责人 →（≥50,000 追加分管领导）财务专员
  - 外包员工 → 甲方项目经理 → 部门负责人 → 财务专员
- 参与角色：申请人、各级审批人、ECS、BMS
- 步骤：SubmitPreApproval →（PreApprovalSubmitted）→ RouteApproval → ApprovePreApproval →（PreApprovalApproved）→ ReserveBudget
- 异常分支：ApprovePreApproval 驳回 →（PreApprovalRejected）→ SubmitPreApproval 重新提交。

### WF-03 报销主流程（端到端，事件驱动）
- 触发事件：员工费用发生并取得发票
- 编排策略：事件驱动 + 分支
- 参与角色：申请人、各级审批人、财务专员、出纳、ECS、IVC、GL-ERP、TPS、DAS
- 步骤：
  1. VerifyInvoice →（InvoiceVerified）
  2. CreateExpenseClaim → AddClaimLineItem → LinkPreApproval → SubmitExpenseClaim →（ClaimSubmitted）
  3. 并行触发：RunComplianceCheck 与 RouteApproval
  4. 审批：ApproveExpenseClaim 逐节点（NodeApproved）→ 全部通过（ClaimApproved）
  5. FinanceReview →（FinanceReviewed）→ 并行 PostToGL（VoucherPosted）与 CreatePaymentOrder
  6. ExecutePayment →（PaymentCompleted）→ SettleBudget 与 ArchiveClaim →（ClaimArchived）
- 分支条件：
  - 合规阻断（BudgetBlocked / 重复 / 伪票）→ 退回申请人；
  - 超标预警（ComplianceAlertRaised）→ 审批人确认后继续。

### WF-04 多级审批路由子流程
- 触发事件：报销单已提交事件（ClaimSubmitted）
- 编排策略：规则驱动动态节点 + 超时升级
- 参与角色：直属上级、部门负责人、财务专员、分管领导、甲方项目经理、ECS 工作流引擎
- 步骤：RouteApproval（依 ApprovalRule 按员工类型与金额阈值生成节点）→ 逐节点 ApproveExpenseClaim
- 超时分支：节点 5 个工作日未处理 → ApprovalEscalated → 升级上一级。
- 驳回分支：任一节点驳回 → ClaimRejected → 回流 WF-03 补充重提或进入 WF-06 申诉。

### WF-05 付款与核销归档流程
- 触发事件：财务复核已通过事件（FinanceReviewed）
- 编排策略：事件驱动 + 分支（对公/对私）
- 参与角色：财务专员、出纳、ECS、TPS、BMS、GL-ERP、DAS
- 分支条件：
  - 对私付款（内勤/研发个人垫付）→ T+3 内支付；
  - 对公付款（外包服务费/设备采购）→ 按供应商账期（默认月结 30 天）支付。
- 步骤：CreatePaymentOrder → ExecutePayment →（PaymentCompleted）→ SettleBudget（BMS 扣减预算）→ ArchiveClaim（DAS 归档）→（ClaimArchived）。

### WF-06 异常与申诉处理流程
- 触发事件：报销单已驳回事件（ClaimRejected）或超时未报销
- 编排策略：分支 + 人工裁决
- 参与角色：申请人、分管领导、ECS
- 分支：
  - 补充重提分支：申请人按驳回记录修改 → SubmitExpenseClaim 回流 WF-03；
  - 申诉分支：RaiseAppeal →（AppealRaised）→ ResolveAppeal（分管领导终审）→（AppealResolved）→ 终结或回流审批；
  - 超时释放分支：费用超 90 天未报销 → SettleBudget 释放预占 →（ReservationReleased）→ SyncBudgetBalance 回退余额。
