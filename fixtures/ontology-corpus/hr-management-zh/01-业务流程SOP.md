# 企业人事管理全流程 — 标准作业程序 (SOP)

**文档编号:** HRM-SOP-001
**版本:** 3.1
**主责部门:** 人力资源运营中心 (HR运营中心)
**生效日期:** 2026-01-01
**复审周期:** 每年
**适用范围:** 集团及各子公司全体正式员工、固定期限员工、试用期员工(不含劳务派遣与实习生,另行规定)。
**关联文档:** `02-业务规则与制度.md` (HRM-POL-002)、`03-系统与数据实体.md` (HRM-SYS-003)、`04-事件与工作流编排.md` (HRM-WF-004)。

---

## 1. 目的与范围

本 SOP 定义企业人力资源管理的端到端标准作业流程,覆盖**入职、任务派遣、绩效考核、薪酬、福利、申请审批、员工手册执行、离职**八大环节。对每一个步骤,均明确标注:执行的**角色/系统(Actor)**、涉及的**业务对象(实体 Objects)**、执行的**系统动作(Action,含消费/产出)**、触发或产生的**事件(Event)**。

权威记录系统(System of Record)如下,数据契约见 `03-系统与数据实体.md`:

- **核心人力系统 (HRIS)** — 主数据:员工档案、岗位、部门、组织架构、劳动合同、薪资项主数据。
- **招聘入职系统 (Onboarding系统)** — 入职单、报到任务、入职材料、工号/账号开通申请。
- **绩效管理系统 (绩效系统)** — 绩效目标、考核周期、360评估、绩效结果、绩效改进计划 (PIP)。
- **薪酬福利系统 (薪酬系统)** — 薪资结构、调薪单、奖金、计税、发薪批次、社保公积金、福利。
- **审批工作流引擎 (审批引擎)** — 各类申请单(请假、加班、调岗、转正、晋升、出差、离职)及多级审批流。
- **考勤系统 (考勤系统)** — 打卡记录、出勤汇总、假期额度、考勤异常。
- **IT账号与资产系统 (IT资产系统)** — 域账号、邮箱、VPN、办公资产领用与回收。

本流程分解为六个子流程(详见 `04-事件与工作流编排.md`):

- **P1 — 入职流程(Offer 到转正)**
- **P2 — 任务派遣与工时(项目/任务分配、外派)**
- **P3 — 绩效考核(目标设定到结果应用)**
- **P4 — 薪酬调整与发放(调薪、奖金、计税、发薪)**
- **P5 — 申请与审批(请假/加班/调岗/晋升/出差等)**
- **P6 — 离职流程(申请到社保停缴与离职面谈)**

## 2. 角色与系统 (Actor 清单)

| 角色 / 系统 | 职责 |

| 用人部门经理 (用人经理) | 发起入职需求、分配任务、设定绩效目标、参与考核、审批下属申请。 |
| 人力资源业务伙伴 (HRBP) | 校验编制、推进入职/转正/调岗/离职、组织绩效校准。 |
| 入职专员 (入职专员) | 生成入职单、收集入职材料、办理劳动合同、发起工号/账号开通。 |
| 薪酬专员 (薪酬专员) | 维护薪资结构、处理调薪单、计算奖金、执行计税与发薪。 |
| 福利专员 (福利专员) | 办理五险一金、补充医疗、假期额度、补贴、员工关怀。 |
| 绩效专员 (绩效专员) | 维护考核周期、发起评估、汇总绩效结果、跟踪 PIP。 |
| 部门负责人 / 分管副总 (部门负责人) | 审批超阈值调薪、晋升、强制分布校准、离职审批。 |
| 直接主管 (直接主管) | 一级审批人;考核评分;工作交接确认。 |
| IT 管理员 (IT管理员) | 开通/回收域账号、邮箱、VPN、办公资产。 |
| 财务出纳 (财务) | 执行发薪打款、个税申报。 |
| 法务专员 (法务) | 审核竞业协议、保密协议、劳动合同模板。 |

## 3. P1 — 入职流程(Offer 到转正)

### 3.1 录用与入职单生成
1. **用人经理** 在招聘环节确认录用后,在 **招聘入职系统** 发起入职需求。系统执行**动作:创建入职单 (CreateOnboardingTicket)**,消费 `候选人` 信息与 `岗位`,产出 `入职单`(状态=`待报到`)。产生事件 **入职单已创建 (OnboardingTicketCreated)**。
2. **入职专员** 在 **招聘入职系统** 执行**动作:发送 Offer (SendOffer)**,消费 `入职单`、`薪资结构`(由薪酬专员预填),产出 `Offer`(状态=`待接受`)。产生事件 **Offer已发出 (OfferSent)**。
3. **新员工** 在线接受 Offer,系统执行**动作:确认 Offer (AcceptOffer)**,将 `Offer` 状态更新为 `已接受`,并写回 `入职单`。产生事件 **Offer已接受 (OfferAccepted)**。
   - **异常路径 — Offer 被拒:** 若新员工在有效期内拒绝,系统将 `Offer` 状态置为 `已拒绝`,`入职单` 置为 `已关闭`,产生事件 **Offer已拒绝 (OfferDeclined)**,流程终止并通知用人经理。
   - **异常路径 — Offer 超时未回:** Offer 发出后超过 7 个自然日未确认,审批引擎自动将其置为 `已失效`,产生事件 **Offer已失效 (OfferExpired)**。

### 3.2 报到与入职材料
4. **新员工** 按报到日到岗,**入职专员** 在 **招聘入职系统** 执行**动作:登记报到 (RegisterCheckin)**,消费 `入职单`,产出 `报到记录`,产生事件 **员工已报到 (EmployeeCheckedIn)**。
5. **新员工** 上传入职材料(身份证、学历证明、原单位离职证明、体检报告、银行卡等)。系统执行**动作:校验入职材料 (ValidateOnboardingDocs)**,消费 `入职材料`,产出校验结果。
   - **异常路径 — 材料缺失:** 若必填材料缺失,系统将 `入职单` 标记为 `材料待补`,产生事件 **入职材料异常 (OnboardingDocsRejected)**,通知新员工限期补交;逾期 5 个工作日未补则触发延期入职审批。

### 3.3 签订劳动合同与建档
6. **入职专员** 在 **核心人力系统** 执行**动作:生成劳动合同 (GenerateLaborContract)**,消费 `入职单`、`岗位`、`薪资结构`,由 **法务** 选定合同模板,产出 `劳动合同`(状态=`待签署`)。
7. **新员工** 与 **HRBP** 双方电子签署,系统执行**动作:签署劳动合同 (SignLaborContract)**,将 `劳动合同` 状态更新为 `已生效`,并创建正式 `员工` 主档(状态=`试用期`)。产生事件 **劳动合同已签署 (LaborContractSigned)** 与 **员工档案已建立 (EmployeeProfileCreated)**。

### 3.4 工号/账号/资产开通
8. **入职专员** 在 **招聘入职系统** 执行**动作:发起账号开通 (RequestAccountProvisioning)**,消费新建 `员工` 主档,向 **IT资产系统** 发出 `账号开通申请`。产生事件 **账号开通已申请 (AccountProvisioningRequested)**。
9. **IT管理员** 在 **IT资产系统** 执行**动作:开通账号与资产 (ProvisionAccount)**,产出 `工号`、`域账号`、`邮箱`、`VPN`、`办公资产领用单`。产生事件 **账号已开通 (AccountProvisioned)**。

### 3.5 试用期目标与转正
10. **直接主管** 在 **绩效系统** 执行**动作:设定试用期目标 (SetProbationGoals)**,消费 `员工`、`岗位`,产出 `绩效目标`(类型=`试用期`)。产生事件 **试用期目标已设定 (ProbationGoalsSet)**。
11. 试用期届满前 15 个工作日,**绩效系统** 自动产生事件 **转正评估到期 (ProbationReviewDue)**。**直接主管** 执行**动作:提交转正评估 (SubmitProbationReview)**,消费试用期 `绩效目标`,产出转正建议。
12. **HRBP** 审批转正建议,系统执行**动作:办理转正 (ConfirmRegularization)**,将 `员工` 状态由 `试用期` 更新为 `正式`,触发薪资由试用期标准切换为正式标准。产生事件 **员工已转正 (EmployeeRegularized)**。
    - **异常路径 — 试用期不通过:** 若转正评估结论为 `不通过`,系统执行**动作:发起试用期解除 (InitiateProbationTermination)**,产生事件 **试用期未通过 (ProbationFailed)**,转入 P6 离职流程(原因=`试用期不合格`)。

## 4. P2 — 任务派遣与工时(项目/任务分配、外派)

13. **用人经理** 在 **核心人力系统** 执行**动作:分配任务 (AssignTask)**,消费 `员工`、`项目`,产出 `任务`(含工时预算、起止日期)。产生事件 **任务已分配 (TaskAssigned)**。
14. **员工** 在 **考勤系统** 执行**动作:填报工时 (LogWorkHours)**,消费 `任务`,产出 `工时` 记录。每周提交一次。
15. **直接主管** 执行**动作:审核工时 (ApproveWorkHours)**,将 `工时` 状态置为 `已确认`,作为绩效与项目核算依据。产生事件 **工时已确认 (WorkHoursApproved)**。
16. 外派场景:**用人经理** 发起 `外派单`(实为 `任务` 子类型,含外派地点、津贴标准、期限),经 **部门负责人** 与 **HRBP** 审批后,**福利专员** 执行**动作:核定外派津贴 (CalculateDispatchAllowance)**,产生事件 **外派已生效 (DispatchActivated)**。
    - **异常路径 — 工时超预算:** 当某 `任务` 累计 `工时` 超过工时预算 20% 时,**考勤系统** 产生事件 **工时超预算告警 (WorkHoursOverBudget)**,通知用人经理调整任务或追加预算。

---

## 5. P3 — 绩效考核(目标设定到结果应用)

### 5.1 目标设定
17. 每个考核周期开始,**绩效系统** 自动开启 `考核周期`(季度/半年/年度),产生事件 **考核周期已开启 (ReviewCycleOpened)**。
18. **用人经理** 与 **员工** 共同执行**动作:设定绩效目标 (SetPerformanceGoals)**,消费 `员工`、`岗位`,产出 `绩效目标`(OKR 或 KPI,含权重)。产生事件 **绩效目标已设定 (PerformanceGoalsSet)**。

### 5.2 评估与 360 反馈
19. 考核周期结束,**绩效系统** 产生事件 **考核周期已结束 (ReviewCycleClosed)**,自动发起评估任务。
20. **员工** 执行**动作:提交自评 (SubmitSelfReview)**;**直接主管** 执行**动作:提交主管评估 (SubmitManagerReview)**;系统按需向同事/下属/客户发起 **360评估** 邀约,执行**动作:收集360评估 (Collect360Feedback)**,消费 `绩效目标` 与多方反馈,产出 `评估`(状态=`待校准`)。产生事件 **评估已提交 (ReviewSubmitted)**。

### 5.3 校准与结果
21. **HRBP** 与 **部门负责人** 召开校准会,执行**动作:执行绩效校准 (CalibratePerformance)**,按强制分布对部门内 `评估` 排序定级,产出 `绩效结果`(等级 S/A/B/C/D)。产生事件 **绩效结果已确定 (PerformanceResultFinalized)**。
22. 系统执行**动作:发布绩效结果 (PublishPerformanceResult)**,将 `绩效结果` 推送给员工本人。产生事件 **绩效结果已发布 (PerformanceResultPublished)**,并向 **薪酬系统** 传递结果以挂钩调薪与奖金。

### 5.4 绩效改进 (PIP)
23. **异常路径 — 绩效不达标:** 当 `绩效结果` 等级为 `D`(或连续两个周期为 `C`)时,系统执行**动作:发起绩效改进计划 (InitiatePIP)**,产出 `绩效改进计划 (PIP)`(周期通常 60 天,含改进目标)。产生事件 **PIP已发起 (PIPInitiated)**。
24. PIP 期满,**直接主管** 执行**动作:评估PIP结果 (EvaluatePIP)**。若达标则关闭 PIP;若不达标产生事件 **PIP未通过 (PIPFailed)**,可依规转入 P6 离职流程(原因=`绩效不胜任`)。

## 6. P4 — 薪酬调整与发放(调薪、奖金、计税、发薪)

25. **薪酬专员**(或绩效结果触发)执行**动作:发起调薪 (InitiateSalaryAdjustment)**,消费 `员工`、`绩效结果`、`薪资结构`,产出 `调薪单`(含调整前/后金额、生效月份)。产生事件 **调薪单已提交 (SalaryAdjustmentSubmitted)**。
26. **调薪单** 按金额阈值进入多级审批(详见 R-12~R-14)。审批通过后系统执行**动作:核准调薪 (ApproveSalaryAdjustment)**,更新 `员工` 的 `薪资结构`。产生事件 **调薪已核准 (SalaryAdjustmentApproved)**。
27. **薪酬专员** 执行**动作:核算奖金 (CalculateBonus)**,消费 `绩效结果` 与奖金池规则,产出 `奖金` 记录。产生事件 **奖金已核算 (BonusCalculated)**。
28. 每月发薪日前,**薪酬系统** 执行**动作:生成发薪批次 (GeneratePayrollBatch)**,消费 `薪资结构`、`奖金`、`工时`、`考勤异常`、`社保公积金`,执行**动作:计算个税 (ComputeIncomeTax)** 产出应发/实发/代扣明细,汇总为 `发薪批次`。产生事件 **发薪批次已生成 (PayrollBatchGenerated)**。
29. **薪酬专员** 复核后提交,**财务** 执行**动作:执行发薪 (DisbursePayroll)**,完成银行代发与个税申报。产生事件 **薪资已发放 (PayrollDisbursed)**。
    - **异常路径 — 发薪批次差错:** 复核发现差错(如考勤扣款异常),系统将 `发薪批次` 退回 `草稿`,产生事件 **发薪批次已驳回 (PayrollBatchRejected)**,薪酬专员修正后重算。

## 7. P5 — 申请与审批(请假/加班/调岗/晋升/出差等)

30. **员工** 在 **审批引擎** 执行**动作:提交申请单 (SubmitRequest)**,消费 `员工`、`假期额度`(请假场景)等,产出统一 `申请单`(类型枚举:`请假`/`加班`/`调岗`/`转正`/`晋升`/`出差`/`离职`)。产生事件 **申请单已提交 (RequestSubmitted)**。
31. **审批引擎** 按申请类型与阈值路由 `审批流`,各级审批人(直接主管 → 部门负责人 → HRBP)依次执行**动作:审批申请单 (ApproveRequest)** 或 **驳回申请单 (RejectRequest)**。
32. 全部审批通过后系统执行**动作:落实申请结果 (ApplyRequestOutcome)**:请假回写 `假期额度`、调岗更新 `员工` 的 `岗位`/`部门`、晋升更新 `岗位` 职级并联动调薪。产生事件 **申请已批准 (RequestApproved)**。
    - **异常路径 — 申请被驳回:** 任一审批人驳回,系统将 `申请单` 置为 `已驳回`,产生事件 **申请已驳回 (RequestRejected)**,通知申请人。
    - **特殊路径 — 晋升联动:** `晋升` 申请批准后自动触发 P4 调薪流程(原因=`晋升调薪`)。

---

## 8. P6 — 离职流程(申请到社保停缴与离职面谈)

33. **员工**(主动离职)或 **HRBP**(被动离职/试用期不合格/绩效不胜任)在 **审批引擎** 执行**动作:发起离职 (InitiateResignation)**,产出 `离职单`(含离职类型、最后工作日)。产生事件 **离职已发起 (ResignationInitiated)**。
34. `离职单` 经 **直接主管 → 部门负责人 → HRBP** 审批通过后,系统执行**动作:生成交接单 (GenerateHandoverTicket)**,产出 `交接单`(列明工作事项、资产、文档、客户)。产生事件 **交接已开始 (HandoverStarted)**。
35. **员工** 完成交接,**直接主管** 执行**动作:确认交接完成 (ConfirmHandover)**,将 `交接单` 置为 `已完成`。产生事件 **交接已完成 (HandoverCompleted)**。
36. **IT管理员** 执行**动作:回收账号与资产 (DeprovisionAccount)**,消费 `办公资产领用单`、`域账号`,产出回收记录。产生事件 **账号已回收 (AccountDeprovisioned)**。
37. **法务** 校验竞业/保密义务,**HRBP** 执行**动作:开具离职证明 (IssueSeparationCertificate)**,产出 `离职证明`。产生事件 **离职证明已开具 (SeparationCertificateIssued)**。
38. **福利专员** 执行**动作:停缴社保公积金 (StopSocialInsurance)**,更新 `社保公积金` 状态为 `已停缴`。产生事件 **社保已停缴 (SocialInsuranceStopped)**。
39. **HRBP** 执行**动作:开展离职面谈 (ConductExitInterview)**,产出离职面谈记录;系统执行**动作:结算末月薪资 (SettleFinalPay)**,触发 P4 末次发薪。完成后将 `员工` 状态置为 `离职`。产生事件 **离职已完成 (SeparationCompleted)**。

---

## 9. 异常与例外处理汇总

| 异常场景 | 触发事件 | 处理动作 |
|---|---|---|
| Offer 超时未确认 | OfferExpired | 自动失效,通知用人经理重新发起 |
| 入职材料缺失 | OnboardingDocsRejected | 限期补交,逾期转延期入职审批 |
| 试用期不通过 | ProbationFailed | InitiateProbationTermination → P6 |
| 工时超预算 | WorkHoursOverBudget | 通知用人经理调整 |
| 绩效等级 D / PIP 未通过 | PIPFailed | 转 P6(绩效不胜任) |
| 发薪批次差错 | PayrollBatchRejected | 退回草稿重算 |
| 申请被驳回 | RequestRejected | 通知申请人 |
