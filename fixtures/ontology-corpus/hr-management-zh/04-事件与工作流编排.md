# 企业人事管理 — 事件目录与工作流编排

**文档编号:** HRM-WF-004
**版本:** 3.1
**主责部门:** 人力资源信息化 (HRIS) & 流程架构组
**生效日期:** 2026-01-01
**关联文档:** `01-业务流程SOP.md` (HRM-SOP-001)、`02-业务规则与制度.md` (HRM-POL-002)、`03-系统与数据实体.md` (HRM-SYS-003)。

本文档定义事件驱动的工作流编排:第 1 节为**事件目录**(每个事件含载荷字段、产生动作、消费动作);第 2 节为**工作流定义**,将动作按顺序/分支/事件驱动串成端到端流程。事件、动作、实体、角色命名与前述文档完全一致。

## 2. 工作流定义 (Workflow Definitions)

### WF-1 — 入职到转正工作流 (Onboarding-to-Regularization)
- **触发事件:** `OnboardingTicketCreated`(用人经理发起入职需求)
- **参与角色:** 用人经理、入职专员、新员工、HRBP、IT管理员、直接主管、法务
- **编排策略:** 顺序为主 + 事件驱动握手 + 异常分支
- **动作链:**
  1. SendOffer → 事件 `OfferSent`
  2. **分支(等待新员工响应):**
     - AcceptOffer → `OfferAccepted` → 进入步骤 3
     - 拒绝 → `OfferDeclined` → 关闭入职单(流程终止)
     - 超时 7 天 → `OfferExpired` → 通知用人经理重发(流程终止)
  3. RegisterCheckin → `EmployeeCheckedIn`
  4. ValidateOnboardingDocs → **分支:** 材料齐 → 步骤 5;材料缺(`OnboardingDocsRejected`)→ 限期补交后回到步骤 4
  5. GenerateLaborContract → SignLaborContract → `LaborContractSigned` + `EmployeeProfileCreated`
  6. RequestAccountProvisioning → `AccountProvisioningRequested` → ProvisionAccount → `AccountProvisioned`
  7. SetProbationGoals → `ProbationGoalsSet`
  8. **定时触发:** `ProbationReviewDue` → SubmitProbationReview
     - **分支:** 达标(满足 R-06)→ ConfirmRegularization → `EmployeeRegularized`(终态)
     - 不达标 → `ProbationFailed` → 移交 **WF-6 离职工作流**

### WF-2 — 任务派遣与工时工作流 (Task-Dispatch-and-Worklog)
- **触发事件:** `EmployeeRegularized` 或用人经理手动发起
- **参与角色:** 用人经理、员工、直接主管、福利专员、部门负责人、HRBP
- **编排策略:** 顺序 + 周期循环(每周工时) + 监控分支
- **动作链:**
  1. AssignTask → `TaskAssigned`(若 `task_type=外派` 且期限 > 30 天,进入外派审批,经审批后 CalculateDispatchAllowance → `DispatchActivated`)
  2. **循环(每周):** LogWorkHours → ApproveWorkHours → `WorkHoursApproved`
  3. **监控分支:** 累计工时超预算 20%(R-36)→ `WorkHoursOverBudget` → 通知用人经理调整任务/追加预算
  4. 工时数据汇入 **WF-4 薪酬工作流** 作为计薪与项目核算依据

### WF-3 — 绩效考核工作流 (Performance-Review)
- **触发事件:** `ReviewCycleOpened`(考核周期定时开启)
- **参与角色:** 用人经理、员工、直接主管、360评估人、HRBP、部门负责人、绩效专员
- **编排策略:** 事件驱动 + 并行收集 + 强制分布校准 + 异常 PIP 分支
- **动作链:**
  1. SetPerformanceGoals → `PerformanceGoalsSet`
  2. **定时触发:** `ReviewCycleClosed` → **并行:** SubmitSelfReview ∥ SubmitManagerReview ∥ Collect360Feedback(需 ≥3 人,R-22)
  3. 全部提交(R-21)→ `ReviewSubmitted` → CalibratePerformance(按强制分布 R-20)→ `PerformanceResultFinalized`
  4. PublishPerformanceResult → `PerformanceResultPublished`(推送 **WF-4**)
  5. **异常分支:** 等级 `D` 或连续两期 `C`(R-23)→ InitiatePIP → `PIPInitiated` → (60 天后)EvaluatePIP
     - 通过 → 关闭 PIP(回归正常)
     - 不通过 → `PIPFailed` → 移交 **WF-6 离职工作流**(原因=`绩效不胜任`)

### WF-4 — 薪酬调整与发放工作流 (Compensation-and-Payroll)
- **触发事件:** `PerformanceResultPublished` / `SalaryAdjustmentApproved` / 每月发薪定时器
- **参与角色:** 薪酬专员、直接主管、部门负责人、HRBP、分管副总、财务
- **编排策略:** 事件驱动 + 阈值分级审批 + 月度批处理 + 复核回退分支
- **动作链:**
  1. InitiateSalaryAdjustment → `SalaryAdjustmentSubmitted`
  2. **分支(按 R-11~R-13 阈值路由审批):**
     - 幅度 ≤ 5% → 直接主管一级审批
     - 5%~15% → 直接主管 + 部门负责人两级
     - > 15% → 直接主管 + 部门负责人 + HRBP 三级 + 副总备案
     - 审批通过 → ApproveSalaryAdjustment → `SalaryAdjustmentApproved`
  3. CalculateBonus(按 R-16 系数)→ `BonusCalculated`
  4. **月度批处理(15 日前):** GeneratePayrollBatch(汇集薪资结构、奖金、工时、考勤扣款、外派津贴、社保)→ ComputeIncomeTax → `PayrollBatchGenerated`
  5. **复核分支:** 复核通过 → DisbursePayroll → `PayrollDisbursed`(25 日前完成);复核驳回 → `PayrollBatchRejected` → 回到步骤 4 重算

### WF-5 — 申请与审批工作流 (Request-and-Approval)
- **触发事件:** `RequestSubmitted`(员工提交任意类型申请单)
- **参与角色:** 员工、直接主管、部门负责人、HRBP
- **编排策略:** 事件驱动 + 按类型/时长动态路由 + 超时自动升级
- **动作链:**
  1. SubmitRequest → `RequestSubmitted`
  2. **动态路由(按 request_type 与阈值):**
     - 请假 ≤1 天 → 直接主管;>1~3 天 → 部门负责人;>3 天 → HRBP(R-30)
     - 加班 → 须事前审批(R-31),直接主管核准
     - 调岗 / 晋升 / 出差 / 离职 → 直接主管 → 部门负责人 → HRBP 多级
  3. **超时分支:** 任一级 >2 工作日未处理(R-38)→ 自动升级上一级
  4. **结果分支:**
     - 全部通过 → ApplyRequestOutcome → `RequestApproved`(请假回写假期额度;调岗回写岗位/部门;晋升回写职级并触发 **WF-4** 晋升调薪)
     - 任一驳回 → RejectRequest → `RequestRejected`(通知申请人)
  5. **特殊衔接:** 当 `request_type=离职` 且批准时,移交 **WF-6 离职工作流**

### WF-6 — 离职工作流 (Separation)
- **触发事件:** `ResignationInitiated`(主动离职 / `ProbationFailed` / `PIPFailed` / 协商解除)
- **参与角色:** 员工、直接主管、部门负责人、HRBP、IT管理员、法务、福利专员、财务
- **编排策略:** 顺序门控 + 多前置条件汇合 + 合规校验
- **动作链:**
  1. InitiateResignation → `ResignationInitiated`(主动离职须满足提前期 R-39)
  2. 多级审批(直接主管 → 部门负责人 → HRBP)通过 → GenerateHandoverTicket → `HandoverStarted`
  3. ConfirmHandover → `HandoverCompleted`(门控:R-40 要求交接完成方可继续)
  4. **并行前置:** DeprovisionAccount → `AccountDeprovisioned` ∥ 法务校验竞业/保密(R-41)
  5. 汇合后:IssueSeparationCertificate → `SeparationCertificateIssued`
  6. StopSocialInsurance(R-42 当月办理)→ `SocialInsuranceStopped`
  7. SettleFinalPay(扣回资产折价与借款 R-43,触发 **WF-4** 末次发薪)+ ConductExitInterview → `SeparationCompleted`(员工状态置 `离职`,终态)

## 3. 编排策略说明

| 工作流 | 主导策略 | 关键分支/网关 |
|---|---|---|
| WF-1 入职到转正 | 顺序 + 事件握手 | Offer 接受/拒绝/超时;材料校验;转正达标判定 |
| WF-2 任务派遣 | 顺序 + 周期循环 | 外派审批;工时超预算监控 |
| WF-3 绩效考核 | 事件驱动 + 并行 | 评估并行收集;强制分布校准;PIP 异常 |
| WF-4 薪酬发放 | 阈值分级 + 批处理 | 调薪幅度三档审批;发薪复核回退 |
| WF-5 申请审批 | 动态路由 | 按类型/时长路由;超时升级;离职衔接 |
| WF-6 离职 | 顺序门控 + 汇合 | 交接门控;账号回收与竞业并行汇合 |
