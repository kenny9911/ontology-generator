## 1. 事件目录

> 每个事件标注: 载荷字段、由哪个【动作】产生、被哪个【动作】消费。动作名称与 `01-业务流程SOP.md` 一致。

### E-01 合同发起请求 (ContractInitiationRequested)
- **载荷:** `合同类型`、`模板ID`、`发起人`
- **产生动作:** (人工触发, 业务发起人)
- **消费动作:** `发起合同` (CreateContract)

### E-02 合同已创建 (ContractCreated)
- **载荷:** `合同ID`、`合同类型`、`状态=草稿`
- **产生动作:** `发起合同` (CreateContract)
- **消费动作:** `校验相对方` (ValidateCounterparty)

### E-03 相对方已校验 (CounterpartyValidated)
- **载荷:** `合同ID`、`相对方ID`、`信用额度`、`黑名单标记=否`
- **产生动作:** `校验相对方` (ValidateCounterparty)
- **消费动作:** `填写条款` (FillClauses)

### E-04 相对方校验不通过 (CounterpartyRejected)
- **载荷:** `合同ID`、`相对方ID`、`阻断原因`
- **产生动作:** `校验相对方` (ValidateCounterparty)
- **消费动作:** `合规裁定` (AdjudicateCompliance)

### E-05 条款已填写 (ClausesFilled)
- **载荷:** `合同ID`、`合同金额`、`税率`、`非标条款清单`
- **产生动作:** `填写条款` (FillClauses)
- **消费动作:** `提交审查` (SubmitForReview)

### E-06 合同已提交审查 (ContractSubmittedForReview)
- **载荷:** `合同ID`、`合同类型`、`合同金额`
- **产生动作:** `提交审查` (SubmitForReview)
- **消费动作:** `执行法务审查` (PerformLegalReview)、`执行财务审查` (PerformFinanceReview)、`评定风险等级` (AssessRiskLevel) —— 并行扇出

### E-07 法务审查完成 (LegalReviewCompleted)
- **载荷:** `合同ID`、`意见ID`、`结论 {通过|带条件通过|驳回}`
- **产生动作:** `执行法务审查` (PerformLegalReview)
- **消费动作:** `路由审批` (RouteApproval)

### E-08 法务审查驳回 (LegalReviewRejected)
- **载荷:** `合同ID`、`修改建议`
- **产生动作:** `执行法务审查` (PerformLegalReview)
- **消费动作:** `填写条款` (FillClauses) —— 退回修改

### E-09 财务审查完成 (FinanceReviewCompleted)
- **载荷:** `合同ID`、`付款计划ID`、`财务结论`
- **产生动作:** `执行财务审查` (PerformFinanceReview)
- **消费动作:** `路由审批` (RouteApproval)

### E-10 信用额度超限 (CreditLimitExceeded)
- **载荷:** `合同ID`、`相对方ID`、`超限金额`
- **产生动作:** `执行财务审查` (PerformFinanceReview)
- **消费动作:** (挂起处理, 人工调额/改预付款)

### E-11 风险等级已评定 (RiskLevelAssessed)
- **载荷:** `合同ID`、`风险等级 {低|中|高}`、`触发合规节点`
- **产生动作:** `评定风险等级` (AssessRiskLevel)
- **消费动作:** `执行合规筛查` (RunComplianceScreening)

### E-12 合规筛查完成 (ComplianceScreeningCompleted)
- **载荷:** `合同ID`、`合规结论`
- **产生动作:** `执行合规筛查` (RunComplianceScreening)
- **消费动作:** `路由审批` (RouteApproval)

### E-13 审批路由已生成 (ApprovalRouteGenerated)
- **载荷:** `合同ID`、`审批流ID`、`审批节点序列`
- **产生动作:** `路由审批` (RouteApproval)
- **消费动作:** `审批节点决策` (DecideApprovalNode)

### E-14 合同审批通过 (ContractApproved)
- **载荷:** `合同ID`、`审批流ID=已通过`
- **产生动作:** `审批节点决策` (DecideApprovalNode) —— 全部节点通过后
- **消费动作:** `发起用印申请` (CreateSealRequest)

### E-15 合同审批驳回 (ContractApprovalRejected)
- **载荷:** `合同ID`、`驳回节点`、`驳回理由`
- **产生动作:** `审批节点决策` (DecideApprovalNode)
- **消费动作:** `填写条款` (FillClauses) —— 退回修改

### E-16 审批超时 (ApprovalTimedOut)
- **载荷:** `合同ID`、`节点ID`、`超时层级`
- **产生动作:** `审批节点决策` (DecideApprovalNode)
- **消费动作:** `审批节点决策` (DecideApprovalNode) —— 催办/升级 (R-32)

### E-17 用印申请已发起 (SealRequestCreated)
- **载荷:** `用印申请ID`、`合同ID`、`用印方式`、`印章ID`
- **产生动作:** `发起用印申请` (CreateSealRequest)
- **消费动作:** `执行盖章` (ApplySeal)

### E-18 合同已盖章 (ContractSealed)
- **载荷:** `合同ID`、`用印申请ID=已盖章`、`签署存证号`
- **产生动作:** `执行盖章` (ApplySeal)
- **消费动作:** `归档合同` (ArchiveContract)

### E-19 签署超时 (SignatureTimedOut)
- **载荷:** `用印申请ID`、`合同ID`
- **产生动作:** `执行盖章` (ApplySeal)
- **消费动作:** `发起用印申请` (CreateSealRequest) —— 退回重发 (R-28)

### E-20 合同已归档 (ContractArchived)
- **载荷:** `合同ID`、`归档台账号`
- **产生动作:** `归档合同` (ArchiveContract)
- **消费动作:** `合同生效` (ActivateContract)

### E-21 合同已生效 (ContractActivated)
- **载荷:** `合同ID`、`生效日期`
- **产生动作:** `合同生效` (ActivateContract)
- **消费动作:** `初始化履约` (InitiatePerformanceTracking)

### E-22 履约已启动 (PerformanceTrackingInitiated)
- **载荷:** `合同ID`、`里程碑清单`、`付款计划ID`
- **产生动作:** `初始化履约` (InitiatePerformanceTracking)
- **消费动作:** `确认里程碑验收` (ConfirmMilestone)

### E-23 里程碑已验收 (MilestoneAccepted)
- **载荷:** `里程碑ID`、`合同ID`、`验收日期`
- **产生动作:** `确认里程碑验收` (ConfirmMilestone)
- **消费动作:** (FSSC 开票动作) → 发出 `发票已开具`

### E-24 发票已开具 (InvoiceIssued)
- **载荷:** `发票ID`、`合同ID`、`金额`、`税额`
- **产生动作:** (FSSC 开票, 由 `里程碑已验收` 触发)
- **消费动作:** `确认里程碑验收` 后续付款跟踪

### E-25 变更单已发起 (ChangeOrderCreated)
- **载荷:** `变更单ID`、`合同ID`、`变更金额`
- **产生动作:** `发起变更单` (CreateChangeOrder)
- **消费动作:** `路由审批` (RouteApproval) —— 变更超 20% 重走审批 (R-36)

### E-26 合同即将到期 (ContractExpiringSoon)
- **载荷:** `合同ID`、`到期日期`、`是否自动续约`
- **产生动作:** `扫描到期合同` (ScanExpiringContracts)
- **消费动作:** `处理续约` (ProcessRenewal)

### E-27 合同已续约 (ContractRenewed)
- **载荷:** `续约ID`、`原合同ID`、`新合同ID`
- **产生动作:** `处理续约` (ProcessRenewal)
- **消费动作:** `初始化履约` (InitiatePerformanceTracking)

### E-28 续约已拒绝 (RenewalDeclined)
- **载荷:** `续约ID`、`原合同ID`
- **产生动作:** `处理续约` (ProcessRenewal)
- **消费动作:** `终止合同` (TerminateContract)

### E-29 合同已终止 (ContractTerminated)
- **载荷:** `合同ID`、`终止结算单号`、`状态 {已终止|已到期}`
- **产生动作:** `终止合同` (TerminateContract)
- **消费动作:** (履约关闭, 档案封存)

## 2. 工作流定义

> 编排策略说明: 【顺序】= 串行步骤; 【并行扇出/汇合】= 多动作并行后等待全部完成; 【事件驱动】= 由事件异步触发; 【分支】= 按条件路由。

### W1 — 合同发起到生效主流程 (端到端)
- **触发事件:** `合同发起请求` (E-01)
- **参与角色:** 业务发起人、法务审查员、财务审查员、风控合规专员、各级审批人、用印管理员、档案管理员
- **编排策略:** 顺序 + 并行汇合 + 分支
- **动作序列:**
  1. `发起合同` → 发出 `合同已创建` (E-02)
  2. `校验相对方` → 分支: `相对方已校验` (E-03) 继续 / `相对方校验不通过` (E-04) 转 W5
  3. `填写条款` → 发出 `条款已填写` (E-05)
  4. `提交审查` → 发出 `合同已提交审查` (E-06), 【并行扇出】至 W2 与 W3
  5. 等待 W2、W3 汇合 (`法务审查完成` E-07 + `财务审查完成` E-09 + `合规筛查完成` E-12 全部就绪)
  6. `路由审批` → `审批节点决策` 逐级 → 分支: `合同审批通过` (E-14) / `合同审批驳回` (E-15) 回到步骤 3
  7. `发起用印申请` → `执行盖章` → 发出 `合同已盖章` (E-18)
  8. `归档合同` → 发出 `合同已归档` (E-20)
  9. `合同生效` → 发出 `合同已生效` (E-21), 启动 W4
- **分支条件:** 审批层级按 R-13/R-14/R-15 (金额) 与 R-23 (高风险加签法务总监)。

### W2 — 法务与财务并行审查子流程
- **触发事件:** `合同已提交审查` (E-06)
- **参与角色:** 法务审查员、财务审查员
- **编排策略:** 并行扇出/汇合
- **动作序列:**
  - 分支 A: `执行法务审查` → `法务审查完成` (E-07) 或 `法务审查驳回` (E-08, 退回 W1 步骤 3)
  - 分支 B: `执行财务审查` → `财务审查完成` (E-09) 或 `信用额度超限` (E-10, 挂起)
- **汇合条件:** 分支 A 与 B 均产出 `完成` 事件后, 交回 W1 的审批路由汇合点。

### W3 — 风控合规审查子流程
- **触发事件:** `合同已提交审查` (E-06)
- **参与角色:** 风控合规专员
- **编排策略:** 顺序 + 分支
- **动作序列:**
  1. `评定风险等级` → 发出 `风险等级已评定` (E-11)
  2. `执行合规筛查` → 分支: `合规筛查完成` (E-12) 交回 W1 汇合 / 命中制裁名单转 W5 `合规裁定`
- **分支条件:** 跨境合同最低风险等级为 `中` (R-24); 高风险合同回写至 W1 审批路由触发法务总监加签 (R-23)。

### W4 — 履约跟踪与开票流程 (事件驱动)
- **触发事件:** `合同已生效` (E-21) 或 `合同已续约` (E-27)
- **参与角色:** 履约跟踪员、财务审查员
- **编排策略:** 事件驱动 + 循环 (按里程碑)
- **动作序列:**
  1. `初始化履约` → 发出 `履约已启动` (E-22)
  2. 对每个里程碑循环: `确认里程碑验收` → 发出 `里程碑已验收` (E-23) → FSSC 开票发出 `发票已开具` (E-24)
  3. 集成/承包合同在每次里程碑付款时扣留质保金 (R-35)
- **分支条件:** 订阅合同按计费周期自动开票 (R-37); 含里程碑合同须有验收标准方可开票 (R-33/R-34)。

### W5 — 变更与续约/终止流程 (事件驱动 + 分支)
- **触发事件:** `变更单已发起` (E-25)、`合同即将到期` (E-26)、`相对方校验不通过` (E-04)、`合规筛查命中`
- **参与角色:** 业务发起人、履约跟踪员、风控合规专员、法务总监、各级审批人
- **编排策略:** 事件驱动 + 多分支
- **动作序列:**
  - 变更分支: `发起变更单` → 若变更超 20% (R-36) → `路由审批` 重走 W1 步骤 6-8
  - 续约分支: `扫描到期合同` → `处理续约` → 分支: `合同已续约` (E-27, 订阅自动续约转 W4) / `续约已拒绝` (E-28) → `终止合同`
  - 阻断/裁定分支: `合规裁定` → `解除阻断` 回 W1 / `终止合同` 发出 `合同已终止` (E-29)
- **分支条件:** 订阅合同到期前 30 天自动续约 (R-37/R-38); 集成/承包合同终止前须完成质保金结算 (R-39); 提前终止须核算违约金 (R-40)。

### W6 — 用印与归档子流程
- **触发事件:** `合同审批通过` (E-14)
- **参与角色:** 业务发起人、用印管理员、档案管理员
- **编排策略:** 顺序 + 异常重试
- **动作序列:**
  1. `发起用印申请` → 发出 `用印申请已发起` (E-17)
  2. `执行盖章` → 分支: `合同已盖章` (E-18) 继续 / `签署超时` (E-19, R-28) 退回步骤 1 重发
  3. `归档合同` (2 个工作日内, R-29) → 发出 `合同已归档` (E-20)
- **分支条件:** 大额或高风险合同必须用法定代表人章或合同专用章 (R-27)。

## 3. 编排总览图 (文字版)

```
[E-01 合同发起请求]
   → 发起合同 →[E-02]→ 校验相对方
        ├─[E-03 已校验]→ 填写条款 →[E-05]→ 提交审查 →[E-06]
        │     ├──(并行)── W2: 执行法务审查 →[E-07] / 执行财务审查 →[E-09]
        │     └──(并行)── W3: 评定风险等级 →[E-11]→ 执行合规筛查 →[E-12]
        │            ↓ (汇合)
        │     路由审批 →[E-13]→ 审批节点决策
        │            ├─[E-14 审批通过]→ W6: 发起用印申请 →[E-17]→ 执行盖章 →[E-18]→ 归档合同 →[E-20]
        │            │        → 合同生效 →[E-21]→ W4: 初始化履约 → 确认里程碑验收 →[E-23]→ 发票已开具 →[E-24]
        │            └─[E-15 审批驳回]→ 退回 填写条款
        └─[E-04 校验不通过]→ W5: 合规裁定 → (解除阻断 / 终止合同 →[E-29])

[E-26 合同即将到期]→ W5: 处理续约 ├─[E-27 已续约]→ W4
                                    └─[E-28 已拒绝]→ 终止合同 →[E-29]
[E-25 变更单已发起]→ W5: 路由审批 (超20%重走审批)
```
